# Codex 模型使用规范

## 1. 文档目的

本文档用于指导本项目在使用网页端 ChatGPT 与 Codex 协作开发时，如何选择模型、如何拆分任务、什么时候新开 Codex 对话，以及如何组织 Codex 提示词。

本项目的基本协作方式是：

```text
网页端 ChatGPT：
负责读取仓库、分析项目现状、拆解任务、判断模型、生成 Codex 提示词。

Codex：
负责执行具体代码修改、运行测试、修复问题、反馈结果。
```

本规范主要给“网页端 ChatGPT”和项目开发者使用。Codex 本身不负责自动切换模型。

---

## 2. 可用模型与定位

本项目主要使用以下模型：

```text
GPT-5.4-Mini
GPT-5.3-Codex
GPT-5.4
GPT-5.5
```

总体选择原则：

```text
简单任务：GPT-5.4-Mini
常规开发：GPT-5.3-Codex
复杂分析：GPT-5.4
高风险评审：GPT-5.5
```

## 4. GPT-5.4-Mini 使用规范

### 4.1 适合场景

GPT-5.4-Mini 用于简单、小范围、低风险、省额度任务。

适合：

```text
1. 修改 README、AGENTS.md、.gitignore。
2. 修改 docs/ 下的普通文档。
3. 修正错别字、格式、标题编号。
4. 补充简单注释或 docstring。
5. 删除未使用 import。
6. 简单变量重命名。
7. 修改日志文案、错误提示文案。
8. 小范围测试名称或断言调整。
9. 明确且低风险的单文件小 bug。
```

### 4.2 不适合场景

不得用 GPT-5.4-Mini 处理：

```text
1. 多文件核心功能实现。
2. 回测、策略、模拟盘、实盘、风控核心逻辑。
3. 数据库结构设计。
4. 复杂 bug 根因分析。
5. 实盘交易相关代码。
6. 资金、订单、持仓、账户、券商接口相关任务。
7. 任何可能影响真实交易安全的任务。
```

### 4.3 提示词模板

```text
【推荐模型】
模型：GPT-5.4-Mini

【选择原因】
本任务是简单、小范围、低风险修改，不涉及核心业务逻辑、实盘交易、资金、订单、持仓或风控，适合使用 GPT-5.4-Mini。

【是否建议新开 Codex 对话】
是

【任务类型】
文档修改 / 简单代码修改

【Codex 提示词】
请完成以下小范围修改：

任务：
...

要求：
1. 只修改与任务直接相关的文件。
2. 不要做额外重构。
3. 不要新增依赖。
4. 不要修改无关功能。
5. 修改完成后输出修改文件列表和简要说明。
```

## 5. GPT-5.3-Codex 使用规范

### 5.1 适合场景

GPT-5.3-Codex 是本项目默认编码模型。

适合：

```text
1. 实现 API 接口。
2. 实现 service、repository、schema、model。
3. 多文件联动修改。
4. 根据 docs/ 中的设计文档落地代码。
5. 实现普通后端功能。
6. 实现回测、策略、模拟盘等模块的普通功能。
7. 补充单元测试或集成测试。
8. 修复普通 bug。
9. 根据 GPT-5.4 或 GPT-5.5 的评审意见执行具体修改。
```

### 5.2 注意事项

GPT-5.3-Codex 可以作为实现模型，但不能作为高风险任务的最终评审模型。

如果任务涉及以下内容：

```text
live
risk
broker
order
trade
account
position
emergency stop
unknown
实盘
资金
风控
下单
撤单
清仓
持仓
券商
```

则 GPT-5.3-Codex 只能负责实现，后续必须安排 GPT-5.4 或 GPT-5.5 复查。

### 5.3 提示词模板

```text
【推荐模型】
模型：GPT-5.3-Codex

【选择原因】
本任务属于常规后端开发或多文件代码实现，需要读取项目结构、修改代码并运行测试，适合使用 GPT-5.3-Codex。

【是否建议新开 Codex 对话】
是 / 否

【任务类型】
常规开发 / 简单代码修改 / 普通 bug 修复

【Codex 提示词】
请先阅读 AGENTS.md 和与本任务相关的 docs/ 文档，不要只依赖 README。

任务目标：
...

要求：
1. 先说明你会阅读哪些文件。
2. 阅读后给出简短修改计划。
3. 按现有项目结构实现，不要新增无关目录。
4. 不要引入不必要的新依赖。
5. 不要删除现有功能。
6. 修改后运行相关测试。
7. 如果测试失败，继续定位并修复。
8. 最后输出：
   - 修改文件列表
   - 主要改动说明
   - 测试命令
   - 测试结果
   - 尚存风险
```

## 6. GPT-5.4 使用规范

### 6.1 适合场景

GPT-5.4 用于复杂分析、方案设计、疑难排错和逻辑复查。

适合：

```text
1. 复杂模块设计。
2. 回测引擎设计或复查。
3. 策略逻辑复查。
4. 风控规则设计或复查。
5. 数据库结构设计。
6. 多模块重构方案。
7. 性能瓶颈分析。
8. GPT-5.3-Codex 连续两轮修复失败后的根因分析。
9. 检查未来函数、数据泄漏、回测与实盘逻辑不一致。
10. 分析模拟盘和实盘交易链路是否一致。
```

### 6.2 使用原则

GPT-5.4 通常用于“分析和设计”，不默认直接大规模改代码。

推荐流程：

```text
1. GPT-5.4 先分析问题或设计方案。
2. GPT-5.4 输出明确任务清单。
3. GPT-5.3-Codex 根据任务清单实现。
4. 必要时 GPT-5.4 再复查实现。
```

### 6.3 提示词模板

```text
【推荐模型】
模型：GPT-5.4

【选择原因】
本任务涉及复杂设计、疑难问题分析或核心逻辑复查，需要较强推理能力，适合使用 GPT-5.4。

【是否建议新开 Codex 对话】
是

【任务类型】
复杂设计 / 疑难排错 / 风控评审

【Codex 提示词】
请先阅读 AGENTS.md 和相关 docs/ 文档。本轮不要急于大规模改代码，先做分析和方案设计。

背景：
...

当前问题：
...

请完成：
1. 阅读相关文件。
2. 分析现有实现或文档中的问题。
3. 找出根因或设计风险。
4. 给出 2-3 个可选方案。
5. 比较每个方案的优缺点。
6. 推荐最稳妥方案。
7. 明确需要修改的文件范围。
8. 输出可交给 GPT-5.3-Codex 执行的任务清单。
9. 如果发现涉及实盘、资金、订单、持仓、风控或券商接口的风险，请标记为高风险，并建议后续使用 GPT-5.5 复审。
```

## 7. GPT-5.5 使用规范

### 7.1 适合场景

GPT-5.5 用于最高风险、最高价值判断场景。

适合：

```text
1. 总体架构评审。
2. 实盘交易上线前最终评审。
3. 资金安全评审。
4. 风控兜底规则评审。
5. 实盘下单、撤单、清仓、强制止损逻辑评审。
6. 订单 unknown 状态处理评审。
7. 重复下单防护评审。
8. 一键停止和异常停机保护评审。
9. GPT-5.4 仍无法判断的高风险疑难问题。
10. 判断某个设计是否会长期影响系统可维护性、安全性和实盘稳定性。
```

### 7.2 不适合场景

GPT-5.5 不用于：

```text
1. 普通编码。
2. 文档修改。
3. 格式调整。
4. 简单 CRUD。
5. 普通测试补充。
6. 简单 bug 修复。
```

### 7.3 提示词模板

```text
【推荐模型】
模型：GPT-5.5

【选择原因】
本任务涉及总体架构、实盘交易、资金安全、风控兜底或上线前最终评审，属于最高风险判断场景，适合使用 GPT-5.5。

【是否建议新开 Codex 对话】
是

【任务类型】
实盘上线前评审 / 风控评审 / 架构评审

【Codex 提示词】
请先阅读 AGENTS.md 和相关 docs/ 文档。本轮作为高风险交易系统评审，不要直接大规模改代码。

项目背景：
本项目是 A 股低频量化交易后端系统，未来会接入实盘交易，不做高频交易。

评审对象：
...

请重点检查：
1. 是否存在真实资金风险。
2. 是否存在绕过风控的路径。
3. 是否存在重复下单风险。
4. 是否存在错误撤单风险。
5. 是否存在错误清仓风险。
6. 是否存在异常加仓风险。
7. 是否存在交易时间控制漏洞。
8. 是否存在行情数据异常导致错误交易。
9. 是否存在回测、模拟盘、实盘逻辑不一致。
10. 是否存在订单状态机错误。
11. 是否存在并发或任务调度风险。
12. 是否存在日志、审计、监控不足。
13. 是否存在测试覆盖不足。
14. 是否存在上线前必须修复的问题。

输出要求：
1. 按严重程度排序。
2. 区分“必须修复 / 建议修复 / 可以暂缓”。
3. 每个问题说明原因。
4. 每个问题给出修复建议。
5. 输出可交给 GPT-5.3-Codex 执行的任务清单。
6. 不要直接大规模修改代码。
```

## 8. 模型升级与降级规则

### 8.1 升级规则

当任务复杂度上升时，按以下路径升级：

```text
GPT-5.4-Mini -> GPT-5.3-Codex -> GPT-5.4 -> GPT-5.5
```

必须升级的情况：

```text
1. GPT-5.4-Mini 开始修改多个无关文件。
2. GPT-5.4-Mini 无法理解项目结构。
3. GPT-5.3-Codex 连续两轮修复失败。
4. 问题涉及架构取舍。
5. 问题涉及实盘、资金、持仓、订单、券商接口。
6. 问题涉及绕过风控、重复下单、unknown 订单、一键停止。
7. 问题可能影响真实交易安全。
```

### 8.2 降级规则

当任务复杂度下降时，可以按以下路径降级：

```text
GPT-5.5 -> GPT-5.4 -> GPT-5.3-Codex -> GPT-5.4-Mini
```

可以降级的情况：

```text
1. 高风险评审已经完成，只剩普通代码实现。
2. 复杂方案已经确定，只剩按清单执行。
3. 功能已经完成，只剩文档、注释或格式修改。
4. bug 根因已经明确，只剩单文件修复。
5. 评审模型已经输出明确任务清单，后续只需要执行。
```

## 9. 新开 Codex 对话规则

### 9.1 建议新开对话的情况

```text
1. 新任务和当前任务无关。
2. 当前对话已经很长。
3. 当前对话中包含多轮失败尝试。
4. 从实现任务切换到架构评审。
5. 从普通开发切换到 GPT-5.5 高风险评审。
6. 需要 GPT-5.4 重新做根因分析。
7. 需要让模型不受上一轮错误方案影响，重新审视问题。
8. 从一个模块切换到另一个独立模块。
```

### 9.2 可以继续当前对话的情况

```text
1. 仍在同一个功能实现任务中。
2. 只是根据刚刚的测试失败继续修复。
3. 只是补充同一任务下遗漏的测试。
4. 只是让 Codex 汇总本轮修改结果。
5. 只是要求 Codex 根据刚才的 diff 做小范围调整。
```

## 10. 高风险任务强制流程

如果任务涉及以下关键词：

```text
live
risk
broker
order
trade
account
position
emergency stop
unknown
实盘
资金
风控
下单
撤单
清仓
持仓
券商
```

必须采用更严格流程：

```text
1. GPT-5.4 做方案设计或逻辑复查。
2. GPT-5.3-Codex 执行代码实现。
3. GPT-5.4 复查实现是否符合设计。
4. GPT-5.5 做最终安全评审。
5. GPT-5.3-Codex 根据评审意见修复。
6. 如风险仍高，GPT-5.5 复核。
```

不得直接使用 GPT-5.4-Mini 处理高风险任务。

不得只用 GPT-5.3-Codex 完成高风险任务后直接合并。

## 11. 网页端 ChatGPT 生成提示词的固定格式

每次让网页端 ChatGPT 生成 Codex 提示词时，必须使用以下格式：

```text
【推荐模型】
模型：GPT-5.4-Mini / GPT-5.3-Codex / GPT-5.4 / GPT-5.5

【选择原因】
说明为什么本任务适合使用该模型。

【是否建议新开 Codex 对话】
是 / 否

【任务类型】
文档修改 / 简单代码修改 / 常规开发 / 复杂设计 / 疑难排错 / 风控评审 / 实盘上线前评审

【Codex 提示词】
从这里开始写给 Codex 的完整执行提示词。
```

不得直接只输出 Codex 提示词正文。

## 12. 推荐工作流

### 12.1 普通开发

```text
1. 网页端 ChatGPT 读取仓库和相关 docs。
2. 网页端 ChatGPT 判断任务类型。
3. 网页端 ChatGPT 推荐 GPT-5.3-Codex。
4. 用户复制提示词给 Codex。
5. Codex 修改代码并运行测试。
6. 用户把结果或报错发回网页端 ChatGPT 复查。
```

### 12.2 复杂模块

```text
1. GPT-5.4 先做设计或方案分析。
2. GPT-5.4 输出任务清单。
3. GPT-5.3-Codex 执行实现。
4. GPT-5.3-Codex 运行测试。
5. GPT-5.4 复查逻辑。
6. 如涉及实盘或资金安全，GPT-5.5 最终评审。
```

### 12.3 高风险实盘任务

```text
1. GPT-5.4 做方案设计或根因分析。
2. GPT-5.3-Codex 实现。
3. GPT-5.4 复查实现。
4. GPT-5.5 做最终安全评审。
5. GPT-5.3-Codex 修复评审问题。
6. GPT-5.5 复核关键风险。
```

## 13. 最终原则

本项目的模型选择原则可以总结为：

```text
省额度小任务：GPT-5.4-Mini
主力编码任务：GPT-5.3-Codex
复杂分析任务：GPT-5.4
高风险评审任务：GPT-5.5
```

对于涉及实盘、资金、订单、持仓、风控、券商接口的任务，必须保守处理，不能为了节省额度使用低能力模型完成关键判断。

# A 股低频量化策略研究、回测、模拟盘与实盘执行系统风控设计说明书

## 1. 引言

### 1.1 文档目的

本文档用于描述“A 股低频量化策略研究、回测、模拟盘与实盘执行系统”的风控设计方案。风控系统是本项目接入 A 股实盘交易的核心安全边界，用于防止策略异常、参数错误、行情异常、账户异常、重复下单、超额下单、不可卖持仓卖出、涨跌停违规下单、非交易时间下单等问题。

本文档主要回答以下问题：

1. 系统需要哪些风控层级？
2. 回测、模拟盘、实盘分别如何风控？
3. 实盘下单前需要检查哪些条件？
4. 什么情况下订单必须被拒绝？
5. 什么情况下策略必须被暂停？
6. 什么情况下系统必须停止全部自动交易？
7. 如何处理异常行情、异常账户、异常订单和异常频率？
8. 风控事件如何记录、告警和追溯？
9. 如何保证系统长期符合低频量化交易定位？

### 1.2 适用范围

本文档适用于以下模块：

```text
回测风控模块
模拟盘风控模块
实盘风控模块
账户风控模块
策略风控模块
订单风控模块
A 股交易规则校验模块
交易频率限制模块
异常处理模块
监控告警模块
审计日志模块
```

本文档不适用于：

```text
高频交易风控
Tick 级交易风控
Level-2 盘口交易风控
做市策略风控
盘口套利风控
高频撤单率风控
毫秒级低延迟风控
```

本项目永久不采用高频交易，因此风控系统重点服务于 **A 股低频交易、日线策略、可选分钟级辅助执行、模拟盘和实盘订单安全控制**。

## 2. 风控系统总体目标

### 2.1 总体目标

风控系统的总体目标是：

> 在策略信号进入模拟盘或实盘执行之前，对账户、策略、订单、行情、交易规则和系统状态进行多层校验，确保任何真实订单在提交券商接口前都满足资金、持仓、价格、数量、时间、频率和风险限制。

### 2.2 具体目标

风控系统需要实现以下目标：

```text
1. 防止超额买入。
2. 防止超额卖出。
3. 防止卖出不可卖持仓。
4. 防止非交易日和非交易时段下单。
5. 防止停牌股票下单。
6. 防止涨跌停价格之外下单。
7. 防止重复下单。
8. 防止策略异常频繁下单。
9. 防止账户仓位过高。
10. 防止单只股票仓位过高。
11. 防止账户亏损超过阈值后继续交易。
12. 防止账户最大回撤超过阈值后继续交易。
13. 防止行情异常时继续交易。
14. 防止券商接口异常时继续交易。
15. 防止订单状态未知时继续交易。
16. 防止风控模块异常时继续交易。
17. 支持一键停止所有自动交易。
18. 记录完整风控事件和审计日志。
```

## 3. 风控设计原则

### 3.1 风控前置原则

所有实盘订单在提交券商接口之前，必须经过风控检查。

```text
策略信号
   ↓
交易计划
   ↓
候选订单
   ↓
风控检查
   ↓
通过：提交券商
拒绝：记录风控事件，不下单
```

策略信号不能绕过风控直接进入订单模块。

### 3.2 风控独立原则

风控模块必须独立于策略模块存在。

策略可以建议：

```text
买什么
卖什么
买多少
卖多少
目标仓位是多少
```

但最终是否允许执行，必须由风控模块决定。

### 3.3 默认保守原则

当系统状态不确定时，风控系统应默认拒绝交易。

例如：

```text
行情状态未知 → 拒绝下单
账户资金未知 → 拒绝下单
持仓可卖数量未知 → 拒绝下单
订单状态未知 → 暂停策略
风控配置缺失 → 拒绝实盘下单
券商接口异常 → 拒绝下单
```

### 3.4 实盘优先安全原则

回测和模拟盘可以允许部分宽松配置，但实盘风控必须严格执行。

例如，回测中可以模拟涨停买入失败，但实盘中如果股票已涨停且买入价格不合理，应直接拒绝订单。

### 3.5 可追溯原则

每一次风控检查都应记录：

```text
检查对象
检查时间
检查规则
检查结果
拒绝原因
系统动作
关联策略
关联订单
关联账户
```

### 3.6 分层风控原则

风控不应只做单一检查，而应分为多个层级：

```text
全局风控
账户级风控
策略级风控
订单级风控
市场状态风控
A 股规则风控
系统状态风控
交易频率风控
```

### 3.7 低频约束原则

本系统永久不做高频交易，因此风控系统必须限制策略异常高频触发。

风控需要监控：

```text
单策略每日信号数
单策略每日订单数
单股票每日交易次数
短时间重复下单
单日换手率
策略执行间隔
```

## 5. 风控层级设计

## 5.1 全局风控

### 5.1.1 说明

全局风控控制整个系统是否允许自动交易，是最高级别风控。

### 5.1.2 检查项

| 编号     | 检查项    | 说明                   | 失败动作 |
| ------ | ------ | -------------------- | ---- |
| GR-001 | 全局实盘开关 | 实盘交易是否启用             | 拒绝下单 |
| GR-002 | 一键停止状态 | 是否已触发 emergency stop | 拒绝下单 |
| GR-003 | 系统运行状态 | API、任务、数据库是否正常       | 拒绝下单 |
| GR-004 | 券商连接状态 | 券商接口是否连接正常           | 拒绝下单 |
| GR-005 | 行情连接状态 | 行情源是否正常              | 拒绝下单 |
| GR-006 | 风控配置状态 | 是否存在有效风控配置           | 拒绝下单 |

### 5.1.3 全局风控规则

```text
如果 global_live_trading_enabled = false，则拒绝所有实盘订单。
如果 emergency_stop = true，则拒绝所有实盘订单。
如果 broker_connection_status != connected，则拒绝所有实盘订单。
如果 market_data_status != normal，则拒绝所有实盘订单。
如果 risk_config_status != active，则拒绝所有实盘订单。
```

## 5.2 账户级风控

### 5.2.1 说明

账户级风控用于控制真实券商账户整体风险，防止账户层面出现过度暴露、过大亏损或超过承受范围的风险。

### 5.2.2 检查项

| 编号     | 检查项      | 说明               | 失败动作      |
| ------ | -------- | ---------------- | --------- |
| AR-001 | 总仓位上限    | 股票市值 / 总资产不得超过阈值 | 拒绝买入      |
| AR-002 | 最低现金比例   | 可用现金不得低于阈值       | 拒绝买入      |
| AR-003 | 单日最大亏损   | 当日亏损超过阈值         | 暂停交易      |
| AR-004 | 最大回撤     | 账户回撤超过阈值         | 停止交易      |
| AR-005 | 单日成交金额上限 | 当日成交金额不得超过阈值     | 拒绝订单      |
| AR-006 | 单日订单数量上限 | 当日订单数不得超过阈值      | 暂停策略或拒绝订单 |
| AR-007 | 账户同步状态   | 账户资金和持仓必须同步成功    | 拒绝下单      |

### 5.2.3 参数示例

```json
{
  "max_total_position": 0.80,
  "min_cash_ratio": 0.20,
  "max_daily_loss_ratio": 0.05,
  "max_account_drawdown": 0.20,
  "max_daily_trade_amount": 500000,
  "max_daily_order_count": 50
}
```

### 5.2.4 账户总仓位计算

```text
total_position_ratio = market_value / total_asset
```

当：

```text
total_position_ratio >= max_total_position
```

系统应拒绝新的买入订单。

### 5.2.5 最低现金比例检查

```text
cash_ratio = available_cash / total_asset
```

如果买入后：

```text
cash_ratio < min_cash_ratio
```

则拒绝买入订单或降低订单金额。

### 5.2.6 账户最大回撤检查

```text
drawdown = (current_total_asset - historical_high_asset) / historical_high_asset
```

当：

```text
drawdown <= -max_account_drawdown
```

系统应：

```text
1. 暂停所有策略。
2. 停止自动买入。
3. 可根据配置生成减仓或清仓建议。
4. 记录 critical 风控事件。
5. 发送告警。
```

## 5.3 策略级风控

### 5.3.1 说明

策略级风控用于限制单个策略实例的资金、仓位、亏损、频率和运行状态。

### 5.3.2 检查项

| 编号     | 检查项     | 说明                | 失败动作 |
| ------ | ------- | ----------------- | ---- |
| SR-001 | 策略状态    | 策略必须处于 running 状态 | 拒绝订单 |
| SR-002 | 策略资金上限  | 单策略使用资金不得超过上限     | 拒绝买入 |
| SR-003 | 策略持仓数上限 | 持仓股票数量不得超过上限      | 拒绝买入 |
| SR-004 | 策略单日亏损  | 策略当日亏损超过阈值        | 暂停策略 |
| SR-005 | 策略最大回撤  | 策略回撤超过阈值          | 暂停策略 |
| SR-006 | 策略连续亏损  | 连续亏损天数超过阈值        | 暂停策略 |
| SR-007 | 策略信号频率  | 信号数超过低频限制         | 暂停策略 |
| SR-008 | 策略订单频率  | 订单数超过低频限制         | 暂停策略 |

### 5.3.3 参数示例

```json
{
  "strategy_capital_limit": 200000,
  "max_hold_stock_count": 10,
  "max_strategy_daily_loss": 0.03,
  "max_strategy_drawdown": 0.15,
  "max_consecutive_loss_days": 5,
  "max_signals_per_day": 50,
  "max_orders_per_day": 20
}
```

### 5.3.4 策略异常频率处理

当策略在短时间内大量生成信号或订单时，系统应认为策略可能异常。

触发条件示例：

```text
单策略每日信号数 > max_signals_per_day
单策略每日订单数 > max_orders_per_day
单股票每日订单数 > max_orders_per_symbol_per_day
信号生成间隔 < min_signal_interval_minutes
```

处理动作：

```text
1. 暂停策略实例。
2. 拒绝后续订单。
3. 记录风控事件。
4. 发送告警。
5. 等待人工确认。
```

## 5.4 订单级风控

### 5.4.1 说明

订单级风控是实盘下单前的最后一道检查，用于保证每一笔订单都合法、可执行、可风控。

### 5.4.2 检查项

| 编号     | 检查项     | 说明                   | 失败动作  |
| ------ | ------- | -------------------- | ----- |
| OR-001 | 股票代码合法  | symbol 必须存在          | 拒绝订单  |
| OR-002 | 方向合法    | side 必须为 buy 或 sell  | 拒绝订单  |
| OR-003 | 价格合法    | price 必须大于 0         | 拒绝订单  |
| OR-004 | 数量合法    | quantity 必须大于 0      | 拒绝订单  |
| OR-005 | 买入资金检查  | 买入金额不得超过可用资金         | 拒绝或缩量 |
| OR-006 | 卖出持仓检查  | 卖出数量不得超过可卖数量         | 拒绝或缩量 |
| OR-007 | 单笔金额上限  | 单笔订单金额不得超过上限         | 拒绝订单  |
| OR-008 | 单票仓位上限  | 买入后单票仓位不得超过上限        | 拒绝或缩量 |
| OR-009 | 重复订单检查  | 不得重复提交同一计划订单         | 拒绝订单  |
| OR-010 | 未完成订单检查 | 同股票同方向已有未完成订单时限制重复下单 | 拒绝或等待 |

### 5.4.3 买入资金检查

订单金额：

```text
order_amount = price * quantity
```

预估成本：

```text
estimated_cost = order_amount + commission + slippage
```

检查：

```text
estimated_cost <= available_cash
```

如果不满足，则：

```text
1. 拒绝订单；
或
2. 根据配置自动缩小下单数量。
```

第一版建议直接拒绝，避免自动缩量造成策略结果不一致。

### 5.4.4 卖出持仓检查

```text
quantity <= available_quantity
```

注意：A 股有 T+1 规则，今日买入数量不能作为今日可卖数量。

### 5.4.5 单笔订单金额限制

```text
order_amount <= max_order_amount
```

超过则拒绝订单。

### 5.4.6 单票仓位限制

买入后单票仓位：

```text
new_symbol_position_ratio =
    (current_symbol_market_value + order_amount) / total_asset
```

检查：

```text
new_symbol_position_ratio <= max_position_per_stock
```

## 5.5 市场状态风控

### 5.5.1 说明

市场状态风控用于控制系统性风险。当大盘处于高风险状态时，系统应降低仓位、暂停买入或清仓。

### 5.5.2 市场状态分类

```text
strong       强势
neutral      中性
weak         弱势
risk_off     高风险
```

### 5.5.3 判断规则示例

```text
沪深300收盘价 > MA60        → strong 或 neutral
沪深300收盘价 < MA60        → weak
MA20 < MA60                 → weak
指数20日回撤超过10%          → risk_off
指数跌破长期均线且放量下跌     → risk_off
```

### 5.5.4 风控动作

| 市场状态     | 风控动作          |
| -------- | ------------- |
| strong   | 正常交易          |
| neutral  | 正常交易，维持仓位限制   |
| weak     | 降低新增仓位，限制买入   |
| risk_off | 停止买入，必要时减仓或清仓 |

### 5.5.5 参数示例

```json
{
  "market_filter_enabled": true,
  "benchmark_symbol": "000300.SH",
  "market_ma_period": 60,
  "risk_off_drawdown": 0.10,
  "weak_market_position_multiplier": 0.5,
  "risk_off_position_multiplier": 0.0
}
```

## 5.6 A 股交易规则风控

### 5.6.1 说明

A 股交易规则风控用于保证订单符合 A 股普通股票交易规则。

### 5.6.2 检查项

| 编号     | 检查项     | 说明              | 失败动作  |
| ------ | ------- | --------------- | ----- |
| CN-001 | 交易日检查   | 非交易日不得下单        | 拒绝订单  |
| CN-002 | 交易时段检查  | 非允许交易时段不得下单     | 拒绝订单  |
| CN-003 | 停牌检查    | 停牌股票不得下单        | 拒绝订单  |
| CN-004 | 涨停检查    | 涨停时买入需限制        | 拒绝或提示 |
| CN-005 | 跌停检查    | 跌停时卖出需限制        | 拒绝或提示 |
| CN-006 | T+1 检查  | 当日买入不可当日卖出      | 拒绝卖出  |
| CN-007 | 买入单位检查  | 买入数量应为 100 股整数倍 | 拒绝或调整 |
| CN-008 | 卖出数量检查  | 卖出不得超过可卖数量      | 拒绝    |
| CN-009 | 价格最小单位  | 价格应符合最小变动单位     | 拒绝或调整 |
| CN-010 | 涨跌停价格范围 | 委托价格不得超出涨跌停范围   | 拒绝订单  |

### 5.6.3 交易日检查

```text
is_trading_day(order_date) == true
```

否则拒绝订单。

### 5.6.4 交易时段检查

第一版可配置允许交易时间：

```text
09:30 - 11:30
13:00 - 15:00
```

是否支持集合竞价由配置决定。

### 5.6.5 停牌检查

```text
is_suspended(symbol, trading_date) == false
```

停牌则拒绝买入和卖出。

### 5.6.6 涨跌停价格检查

买入：

```text
price <= limit_up_price
```

卖出：

```text
price >= limit_down_price
```

价格超出范围拒绝订单。

### 5.6.7 T+1 检查

卖出数量必须满足：

```text
sell_quantity <= available_quantity
```

而不是：

```text
sell_quantity <= total_quantity
```

### 5.6.8 交易单位检查

买入数量：

```text
quantity % 100 == 0
```

卖出数量根据实际规则和券商可卖数量处理，第一版建议严格使用券商返回的 available_quantity。

## 5.7 交易频率风控

### 5.7.1 说明

交易频率风控用于保证系统长期保持低频交易定位，并防止策略异常产生大量信号或订单。

### 5.7.2 检查项

| 编号     | 检查项      | 说明                                 | 失败动作  |
| ------ | -------- | ---------------------------------- | ----- |
| FG-001 | 单策略每日信号数 | 不得超过 max_signals_per_day           | 暂停策略  |
| FG-002 | 单策略每日订单数 | 不得超过 max_orders_per_day            | 暂停策略  |
| FG-003 | 单股票每日订单数 | 不得超过 max_orders_per_symbol_per_day | 拒绝订单  |
| FG-004 | 最短信号间隔   | 信号间隔不得过短                           | 拒绝信号  |
| FG-005 | 单日换手率    | 换手率不得超过阈值                          | 拒绝新订单 |
| FG-006 | 短时间重复下单  | 同股票同方向短时间重复下单                      | 拒绝订单  |

### 5.7.3 参数示例

```json
{
  "max_signals_per_day": 50,
  "max_orders_per_day": 20,
  "max_orders_per_symbol_per_day": 2,
  "min_signal_interval_minutes": 60,
  "max_daily_turnover_rate": 0.5
}
```

### 5.7.4 异常频率处理

当检测到异常高频行为：

```text
1. 拒绝当前订单。
2. 暂停策略实例。
3. 标记风险等级为 critical。
4. 记录风控事件。
5. 发送告警。
6. 等待人工恢复。
```

## 5.8 系统状态风控

### 5.8.1 说明

系统状态风控用于检查运行环境是否允许交易。

### 5.8.2 检查项

| 编号      | 检查项      | 失败动作    |
| ------- | -------- | ------- |
| SYS-001 | 数据库连接异常  | 拒绝交易    |
| SYS-002 | Redis 异常 | 拒绝交易或降级 |
| SYS-003 | 异步任务异常   | 暂停相关任务  |
| SYS-004 | 行情源异常    | 拒绝交易    |
| SYS-005 | 券商接口异常   | 拒绝交易    |
| SYS-006 | 账户同步超时   | 拒绝交易    |
| SYS-007 | 持仓同步超时   | 拒绝交易    |
| SYS-008 | 风控配置缺失   | 拒绝交易    |
| SYS-009 | 风控引擎异常   | 拒绝交易    |

### 5.8.3 默认动作

系统状态异常时，默认动作是：

```text
拒绝新订单，不自动恢复实盘交易。
```

恢复交易必须满足：

```text

账户重新同步成功
持仓重新同步成功

人工或系统恢复标记完成
```

## 6. 回测风控设计

   ↓

## 7. 模拟盘风控设计

### 7.1 说明

模拟盘风控介于回测和实盘之间，用于验证策略在真实时间推进下的运行情况。

### 7.2 模拟盘风控内容

模拟盘应执行与实盘接近的风控：

```text
模拟账户资金检查
模拟持仓可卖数量检查
涨跌停检查
停牌检查
交易时段检查
策略频率限制
单票仓位限制
总仓位限制
止损止盈
异常行情处理
```

### 7.3 模拟盘与实盘差异

| 项目   | 模拟盘     | 实盘       |
| ---- | ------- | -------- |
| 资金   | 虚拟资金    | 真实资金     |
| 订单   | 模拟订单    | 券商真实订单   |
| 成交   | 模拟成交    | 券商成交回报   |
| 风控   | 应尽量接近实盘 | 必须严格执行   |
| 异常处理 | 记录和暂停   | 记录、暂停、告警 |

模拟盘通过后，策略才允许进入实盘候选阶段。

## 8. 实盘风控设计

### 8.1 说明

实盘风控是系统最重要的安全模块。所有真实订单必须经过实盘风控。

### 8.2 实盘风控流程

```text
候选订单
   ↓
全局开关检查
   ↓
策略状态检查
   ↓
账户状态检查
   ↓
行情状态检查
   ↓
交易日检查
   ↓
交易时段检查
   ↓
股票状态检查
   ↓
涨跌停检查
   ↓
T+1 检查
   ↓
资金和持仓检查
   ↓
仓位限制检查
   ↓
订单金额检查
   ↓
交易频率检查
   ↓
生成风控结果
   ↓
通过：进入订单提交
拒绝：记录事件，不下单
```

### 8.3 实盘下单前必须满足

```text
1. 实盘交易开关已开启。
2. 未触发一键停止。
3. 策略实例处于 running 状态。
4. 当前日期为交易日。

6. 行情源状态正常。
7. 券商接口连接正常。
8. 账户资金同步成功。
9. 持仓同步成功。
10. 股票未停牌。
11. 委托价格在涨跌停范围内。
12. 买入金额不超过可用资金。
13. 卖出数量不超过可卖数量。
14. 买入数量符合交易单位规则。
15. 未超过单票仓位限制。
16. 未超过账户总仓位限制。
17. 未超过策略频率限制。
18. 未存在重复订单风险。
```

## 12. 一键停止机制设计

### 12.1 说明

一键停止是实盘交易系统必须具备的最高优先级风控能力。

### 12.2 触发方式

```text
人工点击或调用接口
系统自动触发
账户风险触发
策略异常触发
接口异常触发
```

### 12.3 一键停止动作

触发后系统应：

```text
1. 设置 global_trading_status = stopped。
2. 停止所有策略生成新交易计划。
3. 拒绝所有新订单。
4. 暂停所有自动下单任务。
5. 记录审计日志。
6. 记录 critical 风控事件。
7. 发送告警。
```

### 12.4 是否自动撤单

一键停止是否自动撤销未成交订单应由配置决定。

配置项：

```json
{
  "emergency_stop_cancel_open_orders": false
}
```

第一版建议默认不自动撤单，而是：

```text
停止新下单
同步未完成订单
通知人工处理是否撤单
```

原因是自动撤单可能在某些情况下引入新的交易风险。

### 12.5 恢复条件

恢复自动交易必须满足：

```text
1. 人工确认。
2. 行情源正常。
3. 券商接口正常。
4. 账户同步成功。
5. 持仓同步成功。
6. 未完成订单状态确认。
7. 风控配置有效。
8. 审计日志记录恢复操作。
```

## 13. 重复下单防护设计

### 13.1 重复下单风险

重复下单可能来源于：

```text
接口重试
任务重复执行
程序重启
网络超时
券商返回延迟
策略重复生成信号
交易计划重复执行
```

### 13.2 防护机制

系统应使用以下机制：

```text
1. client_order_id 唯一。
2. trading_plan_id 唯一。
3. signal_id 关联。
4. 同一交易计划只能执行一次。
5. 同一策略、同一股票、同一方向、短时间内限制重复订单。
6. 下单前检查本地未完成订单。
7. 程序重启后先同步券商委托和成交。
```

### 13.3 本地订单号设计

```text
client_order_id = strategy_instance_id + plan_id + symbol + side + date + hash
```

该字段必须在数据库中建立唯一索引。

### 13.4 重试策略

如果下单接口超时：

```text
不能立即再次提交相同订单。
```

应先：

```text
1. 查询本地订单状态。
2. 查询券商当日委托。
3. 查询券商当日成交。
4. 判断订单是否已经提交成功。
5. 无法确认时标记 unknown。
```

## 14. 订单状态未知处理

### 14.1 触发场景

订单状态未知可能发生于：

```text
下单请求超时
券商接口断开
程序崩溃
网络中断
券商返回异常
订单提交成功但本地未收到结果
```

### 14.2 处理原则

订单状态未知时，系统不得继续自动下单。

### 14.3 处理流程

```text
发现订单状态未知
   ↓
标记订单为 unknown
   ↓
暂停相关策略
   ↓
查询券商当日委托
   ↓
查询券商当日成交
   ↓
尝试匹配本地订单
   ↓
如果匹配成功，修正状态
   ↓
如果仍无法确认，保持 unknown
   ↓
发送告警
   ↓
等待人工处理
```

### 14.4 风控动作

```text
1. 暂停策略。
2. 禁止新订单。
3. 记录 critical 风控事件。
4. 记录审计日志。
```

## 17. 实盘风控与交易计划的关系

### 17.1 交易计划不是订单

交易计划只是策略层面的建议，不等于真实订单。

```text
策略信号 → 交易计划 → 候选订单 → 风控检查 → 实盘订单
```

### 17.2 交易计划预风控

收盘后生成交易计划时，可先做预风控：

```text
仓位是否超限
股票是否停牌
是否可能涨跌停
目标仓位是否合理
策略频率是否合理
```

预风控通过并不代表次日一定能下单。

### 17.3 下单前实时风控

实盘下单前必须重新检查：

```text
最新行情
账户资金
持仓可卖数量
订单价格
交易时段
券商状态
```

因为这些信息可能已经变化。

## 18. 风控与仓位管理的关系

### 18.1 仓位管理职责

仓位管理负责计算：

```text
目标买入金额
目标卖出数量
目标持仓比例
```

### 18.2 风控职责

风控负责判断：

```text
是否允许买
是否允许卖
是否超过资金
是否超过仓位
是否超过频率
是否违反规则
```

### 18.3 自动缩量问题

当目标订单超过限制时，有两种处理方式：

```text
1. 拒绝订单。
2. 自动缩小订单数量。
```

第一版建议默认拒绝订单，并记录原因。

后续可以在配置明确允许时启用自动缩量：

```json
{
  "allow_auto_resize_order": true
}
```

但自动缩量必须记录日志，避免策略预期和真实执行不一致。

## 23. 风控验收标准

### 23.1 基础验收

系统应满足：

```text
1. 风控配置可查询、可更新。
2. 风控配置变更有审计日志。
3. 风控检查结果可记录。
4. 风控拒绝订单不会提交券商接口。
5. 风控通过订单可以进入订单管理模块。
```

### 23.2 A 股规则验收

系统应满足：

```text
1. 非交易日拒绝下单。
2. 非交易时段拒绝下单。
3. 停牌股票拒绝下单。
4. 超出涨跌停价格拒绝下单。
5. 买入数量不符合 100 股整数倍时拒绝或调整。
6. 卖出数量超过可卖数量时拒绝。
7. T+1 不可卖持仓不能卖出。
```

### 23.3 账户风控验收

系统应满足：

```text
1. 买入金额超过可用资金时拒绝。
2. 买入后总仓位超过上限时拒绝。
3. 买入后单票仓位超过上限时拒绝。
4. 账户回撤超过阈值时暂停交易。
5. 单日亏损超过阈值时暂停交易。
```

### 23.4 策略风控验收

系统应满足：

```text
1. 策略暂停后不得继续下单。
2. 策略订单数超过限制时暂停。
3. 策略信号频率异常时暂停。
4. 策略连续亏损超过阈值时暂停。
```

### 23.5 异常风控验收

系统应满足：

```text
1. 行情异常时拒绝下单。
2. 券商接口异常时拒绝下单。
3. 账户同步失败时拒绝下单。
4. 持仓同步失败时拒绝下单。
5. 订单状态未知时暂停策略。
6. 一键停止后拒绝所有新订单。
```

## 26. 风控与低频定位的关系

本系统永久不做高频交易，因此风控系统需要持续约束交易频率。

### 26.1 不允许的行为

```text
1. Tick 级信号触发。
2. 秒级或毫秒级重复下单。
3. 高频撤单。
4. 盘口套利。
5. 做市报价。
6. 短时间大量订单。
7. 高频参数优化结果直接实盘。
```

### 26.2 允许的低频行为

```text
1. 日线收盘后生成交易计划。
2. 次日开盘或盘中低频执行。
3. 每日调仓。
4. 每周调仓。
5. 每月调仓。
6. 低频分钟级辅助确认。
```

### 26.3 风控约束

```text
策略 frequency 必须为 daily / weekly / monthly / low_frequency_minute。
超过频率限制时暂停策略。
订单数量超过限制时拒绝订单。
异常信号密度触发 critical 风控事件。
```

## 27. 总结

本文档完成了 A 股低频量化策略研究、回测、模拟盘与实盘执行系统的风控设计。

风控系统的核心定位是：

> 作为策略信号进入实盘订单之前的强制安全边界，保证任何真实订单都必须经过账户、策略、订单、行情、A 股规则、交易频率和系统状态的多层检查。

本文档重点设计了：

```text
1. 全局风控
2. 账户级风控
3. 策略级风控
4. 订单级风控
5. 市场状态风控
6. A 股交易规则风控
7. 交易频率风控
8. 系统状态风控
9. 回测风控
10. 模拟盘风控
11. 实盘风控
12. 一键停止机制
13. 重复下单防护
14. 订单状态未知处理
15. 行情异常处理
16. 账户同步异常处理
17. 风控配置
18. 风控事件
19. 风控接口
20. 风控测试和验收标准
```

本系统虽然不做高频交易，但只要接入 A 股实盘，就必须严格执行风控。风控模块不得被策略绕过，不得被实盘订单绕过，不得在异常状态下默认放行。

最终，风控系统应保证：

```text
能买的不一定买，
不能买的一定不买；
能卖的不一定卖，
不能卖的一定不卖；
状态不确定时，一律不自动交易。
```

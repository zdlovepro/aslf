# A 股低频量化策略研究、回测、模拟盘与实盘执行系统回测引擎设计说明书

## 1. 引言

### 1.1 文档目的

本文档用于描述“A 股低频量化策略研究、回测、模拟盘与实盘执行系统”的回测引擎设计方案。回测引擎是策略研究阶段的核心模块，用于在历史行情数据上模拟策略运行过程，验证策略在历史环境下的收益、风险、交易成本、仓位变化和稳定性。

本文档主要回答以下问题：

```text
1. 回测引擎的总体架构如何设计？
2. 回测如何按交易日推进？
3. 策略信号如何转化为模拟订单？
4. 模拟成交如何处理？
5. A 股 T+1、涨跌停、停牌、交易单位如何模拟？
6. 手续费、印花税、滑点如何计算？
7. 持仓、现金、市值和总资产如何更新？
8. 回测结果如何统计？
9. 如何防止未来函数？
10. 如何保证回测逻辑和实盘逻辑尽量一致？
```

## 2. 回测引擎定位

### 2.1 系统定位

回测引擎用于在历史数据中模拟低频策略交易过程。

本系统回测引擎长期定位为：

```text
A 股低频策略回测引擎
```

支持：

```text
日线级回测
可选分钟级低频回测
A 股交易规则模拟
交易成本模拟
仓位管理模拟
风控规则模拟
参数优化调用
回测报告生成
```

不支持：

```text
Tick 级回测
Level-2 盘口回测
逐笔成交回测
订单簿排队模拟
做市策略回测
高频撤单回测
毫秒级撮合模拟
高频交易延迟建模
```

### 2.2 回测引擎目标

回测引擎的目标是：

> 使用历史数据，在尽可能接近真实 A 股低频交易规则的前提下，模拟策略从生成信号、下单、成交、持仓变化到账户资产变化的全过程，并输出可用于策略评估和参数优化的结果。

### 2.3 回测引擎不保证事项

回测结果不代表未来收益。

回测引擎不能保证：

```text
策略未来一定盈利
实盘成交一定等于回测成交
滑点一定等于设置值
涨跌停时一定能成交
停牌股票能及时卖出
样本内最优参数在未来仍然最优
```

回测只能用于策略验证、对比和风险分析。

## 3. 回测设计原则

### 3.1 低频原则

回测引擎只服务低频策略。

默认以日线为核心粒度：

```text
T 日收盘后生成信号
T+1 日开盘或指定价格执行
```

可扩展低频分钟级执行，但不做 Tick 级撮合。

### 3.2 防未来函数原则

策略在任意交易日只能使用当前交易日及之前的数据。

默认推荐：

```text
T 日收盘后产生信号
T+1 日执行交易
```

不允许：

```text
使用 T+1 日价格决定 T 日是否买入
使用未来最高价计算当前止盈
使用未来最低价计算当前止损
使用未来收益作为当前因子
使用未来成分股或未来停牌状态
```

### 3.3 规则一致原则

回测应尽量模拟实盘规则：

```text
A 股交易日
交易时间
停牌
涨跌停
T+1
买入 100 股整数倍
卖出不得超过可卖数量
手续费
印花税
滑点
最低手续费
```

### 3.4 策略与成交分离原则

策略只生成信号和目标仓位。

回测引擎负责：

```text
信号接收
交易计划生成
模拟订单生成
风控检查
模拟成交
账户更新
结果统计
```

### 3.5 可复现原则

相同输入应得到相同回测结果。

相同输入包括：

```text
策略代码版本
策略参数版本
股票池
回测区间
初始资金
交易成本配置
成交价格规则
风控配置
行情数据版本
```

### 3.6 可追溯原则

回测中每一笔交易都应能追溯到：

```text
策略
参数版本
信号
交易原因
订单
成交价格
成交数量
手续费
滑点
持仓变化
账户资产变化
```

## 7. 回测时间推进机制

### 7.1 日线回测时间推进

日线回测以交易日为单位推进。

```text
for trade_date in trading_days:
    更新当前日期
    更新持仓市值
    检查已有持仓风控
    策略生成信号
    生成订单
    模拟成交
    更新账户和持仓
    记录每日资产
```

### 7.2 推荐默认执行方式

推荐默认使用：

```text
T 日收盘后生成信号
T+1 日开盘价成交
```

原因：

```text
1. 避免使用未来价格。
2. 更接近低频策略实盘执行。
3. 可以减少未来函数风险。
```

### 7.3 支持的执行价格类型

| price_type    | 说明                    |
| ------------- | --------------------- |
| next_open     | 下一交易日开盘价成交            |
| next_close    | 下一交易日收盘价成交            |
| current_close | 当前交易日收盘价成交，仅适合明确收盘前信号 |
| vwap          | 成交量加权价格，第一版可不实现       |
| fixed_price   | 指定价格成交                |

第一版推荐：

```text
next_open
```

### 7.4 时间推进示例

```text
2024-01-02 收盘后：
    计算指标
    策略生成买入信号

2024-01-03 开盘：
    回测引擎按开盘价模拟买入

2024-01-03 收盘：
    更新持仓市值和账户资产
```

## 8. 数据加载设计

### 8.1 数据加载范围

回测数据加载器负责加载：

```text
股票基础信息
交易日历
日线行情
可选分钟行情
技术指标
因子数据
指数行情
停牌状态
涨跌停价格
复权因子
```

### 8.2 数据预加载

为了提高效率，回测任务开始前可以一次性加载所需数据到内存。

适合数据规模：

```text
100 只股票
5 年日线数据
基础指标和因子
```

如果数据规模扩大，可改为分段加载或按日期懒加载。

### 8.3 数据对齐

不同股票可能存在：

```text
上市日期不同
停牌日期不同
缺失行情
退市情况
```

系统需要按交易日历进行数据对齐。

### 8.4 缺失数据处理

缺失数据处理规则：

| 情况     | 处理方式          |
| ------ | ------------- |
| 非交易日   | 跳过            |
| 股票停牌   | 不交易，持仓用最近价格估值 |
| 上市前无数据 | 不参与交易         |
| 退市后无数据 | 不参与交易         |
| 某日价格缺失 | 不生成交易，记录数据异常  |
| 指标不足窗口 | 不生成信号         |

### 8.5 复权处理

回测建议使用前复权价格计算收益和指标。

可配置：

```text
qfq    前复权
hfq    后复权
none   不复权
```

第一版推荐：

```text
qfq
```

## 9. 账户模型设计

### 9.1 回测账户字段

```text
initial_cash          初始资金
cash                  当前现金
frozen_cash           冻结现金，回测可简化
market_value          持仓市值
total_value           总资产
available_cash        可用现金
daily_pnl             当日盈亏
total_pnl             累计盈亏
max_total_value       历史最高资产
drawdown              当前回撤
```

### 9.2 总资产计算

```text
total_value = cash + market_value
```

### 9.3 持仓市值计算

```text
market_value = Σ(position_quantity * current_price)
```

### 9.4 当前回撤计算

```text
drawdown = (total_value - max_total_value) / max_total_value
```

### 9.5 现金更新

买入时：

```text
cash = cash - trade_amount - commission - slippage_cost
```

卖出时：

```text
cash = cash + trade_amount - commission - tax - slippage_cost
```

## 10. 持仓模型设计

### 10.1 持仓字段

每只股票的持仓应包含：

```text
symbol
total_quantity
available_quantity
today_buy_quantity
frozen_quantity
cost_price
market_price
market_value
unrealized_pnl
realized_pnl
holding_days
highest_price_since_buy
last_buy_date
last_sell_date
```

### 10.2 T+1 可卖数量设计

A 股普通股票 T+1，今日买入不能今日卖出。

持仓中需要区分：

```text
total_quantity
available_quantity
today_buy_quantity
```

买入后：

```text
today_buy_quantity += buy_quantity
total_quantity += buy_quantity
available_quantity 不增加
```

下一个交易日开始：

```text
available_quantity += previous_day_today_buy_quantity
today_buy_quantity = 0
```

### 10.3 成本价更新

买入后平均成本：

```text
new_cost_price =
    (old_cost_price * old_quantity + buy_price * buy_quantity + buy_cost)
    / new_total_quantity
```

卖出不改变剩余持仓成本价，或者按移动加权平均规则保持原成本价。

### 10.4 持仓天数

如果持仓数量大于 0：

```text
holding_days += 1
```

如果持仓清空：

```text
holding_days = 0
highest_price_since_buy = null
```

### 10.5 最高价记录

用于移动止盈：

```text
highest_price_since_buy = max(highest_price_since_buy, current_high_or_close)
```

### 11.1 策略信号类型

回测引擎支持以下信号：

```text

sell

```

### 11.2 策略信号字段

```text
signal_id
strategy_id
params_id
symbol
signal_date
signal_type
signal_strength
target_position
reason
source_mode = backtest
```

### 11.3 信号处理流程

```text
策略生成信号
   ↓
信号去重
   ↓
信号合法性检查
   ↓
仓位管理转换为目标交易
   ↓
生成模拟订单
```

### 11.4 信号去重规则

同一回测任务中：

```text
strategy_id + params_id + symbol + signal_date + signal_type
```

应避免重复信号。

### 11.5 信号优先级

当同一股票同一日期出现多个信号时，优先级建议：

```text
clear > sell > reduce > buy > hold > watch
```

即清仓信号优先级最高。

## 12. 订单生成设计

### 12.1 模拟订单字段

```text
order_id
task_id
signal_id
symbol
side
price
quantity
order_type
order_status
created_date
executed_date
reason
```

### 12.2 订单方向

```text
buy
sell
```

### 12.3 订单状态

```text
created
risk_checked
filled
partially_filled
rejected
canceled
failed
```

回测中一般不需要完整实盘订单状态，但应保留基本状态，方便和实盘模型统一。

### 12.4 目标仓位转订单

策略输出目标仓位后，计算当前仓位和目标仓位差额。

当前股票市值：

```text
current_position_value = current_quantity * current_price
```

目标股票市值：

```text
target_position_value = total_value * target_position
```

差额：

```text
delta_value = target_position_value - current_position_value
```

如果：

```text
delta_value > 0
```

生成买入订单。

如果：

```text
delta_value < 0
```

生成卖出订单。

### 12.5 金额转股数

买入数量：

```text
raw_quantity = available_buy_amount / price
```

A 股买入数量取 100 股整数倍：

```text
quantity = floor(raw_quantity / 100) * 100
```

如果 quantity <= 0，则不生成订单。

### 12.6 卖出数量

卖出数量：

```text
quantity = min(target_sell_quantity, available_quantity)
```

如果 quantity <= 0，则不生成卖出订单。

## 13. 回测风控设计

### 13.1 回测风控职责

回测风控负责在模拟订单成交前检查订单是否符合规则。

### 13.2 回测风控检查项

```text
是否交易日
是否停牌
是否涨停买入
是否跌停卖出
是否满足 T+1
买入资金是否足够
卖出可卖数量是否足够
买入数量是否符合 100 股整数倍
是否超过单票仓位
是否超过总仓位
是否低于最低现金保留
是否触发账户最大回撤
```

### 13.3 回测风控流程

```text
模拟订单
   ↓
交易日检查
   ↓
停牌检查
   ↓
涨跌停检查
   ↓
T+1 检查
   ↓
资金/持仓检查
   ↓
仓位限制检查
   ↓
通过 / 拒绝
```

### 13.4 风控拒绝原因

常见拒绝原因：

```text
not_trading_day
stock_suspended
limit_up_cannot_buy
limit_down_cannot_sell
cash_not_enough
position_not_enough
t_plus_one_restricted
quantity_too_small
position_limit_exceeded
total_position_limit_exceeded
min_cash_ratio_violation
```

## 14. A 股交易规则模拟

### 14.1 交易日规则

回测只在交易日推进。

```text
if not is_trading_day(date):
    skip
```

### 14.2 停牌规则

停牌股票：

```text
不能买入
不能卖出
持仓保留
估值使用最近可用收盘价
```

### 14.3 涨停规则

如果模拟买入价格 >= 涨停价，默认认为买入可能失败。

配置项：

```json
{
  "allow_buy_limit_up": false
}
```

第一版建议：

```text
涨停不买入
```

### 14.4 跌停规则

如果模拟卖出价格 <= 跌停价，默认认为卖出可能失败。

配置项：

```json
{
  "allow_sell_limit_down": false
}
```

第一版建议：

```text
跌停不卖出
```

### 14.5 T+1 规则

今日买入股票不能当日卖出。

回测中需要维护：

```text
today_buy_quantity
available_quantity
```

### 14.6 交易单位规则

买入数量必须为 100 股整数倍。

```text
quantity % 100 == 0
```

### 14.7 价格最小变动单位

A 股价格最小变动单位通常为 0.01 元。

回测中可检查：

```text
price 精度为 0.01
```

### 14.8 卖出数量规则

卖出数量不得超过可卖数量。

```text
sell_quantity <= available_quantity
```

## 15. 模拟成交设计

### 15.1 模拟撮合器职责

模拟撮合器负责将模拟订单转化为模拟成交。

职责包括：

```text
确定成交价格
确定成交数量
计算交易成本
判断是否成交
生成交易记录
更新订单状态
```

### 15.2 支持的成交模式

| 模式                   | 说明            |
| -------------------- | ------------- |
| full_fill            | 全部成交          |
| reject_if_untradable | 不可交易时拒绝       |
| partial_fill         | 部分成交，第一版可不实现  |
| volume_limit_fill    | 按成交量限制成交，后续扩展 |

第一版建议：

```text
可交易则全部成交
不可交易则拒绝
```

### 15.3 成交价格

根据 execution_config.price_type 决定：

```text
next_open       下一交易日开盘价
next_close      下一交易日收盘价
current_close   当前交易日收盘价
fixed_price     指定价格
```

### 15.4 滑点处理

买入滑点：

```text
actual_buy_price = price * (1 + slippage_rate)
```

卖出滑点：

```text
actual_sell_price = price * (1 - slippage_rate)
```

### 15.5 成交金额

```text
trade_amount = actual_price * quantity
```

### 15.6 订单状态更新

如果成交成功：

```text
order_status = filled
```

如果风控拒绝：

```text
order_status = rejected
```

如果停牌、涨跌停不能成交：

```text
order_status = rejected
```

## 16. 交易成本设计

### 16.1 成本组成

A 股回测交易成本包括：

```text
买入佣金
卖出佣金
印花税
滑点成本
最低手续费
```

### 16.2 买入成本

```text
commission = max(trade_amount * buy_commission_rate, min_commission)
tax = 0
slippage_cost = trade_amount * slippage_rate
total_cost = commission + slippage_cost
```

### 16.3 卖出成本

```text
commission = max(trade_amount * sell_commission_rate, min_commission)
tax = trade_amount * stamp_tax_rate
slippage_cost = trade_amount * slippage_rate
total_cost = commission + tax + slippage_cost
```

### 16.4 成本配置示例

```json
{
  "buy_commission_rate": 0.0003,
  "sell_commission_rate": 0.0003,
  "stamp_tax_rate": 0.0005,
  "slippage_rate": 0.001,
  "min_commission": 5
}
```

### 16.5 注意事项

交易成本会显著影响低频策略结果，尤其是：

```text
高换手策略
均值回归策略
短持仓周期策略
小资金策略
```

参数优化时必须加入换手率惩罚和成本模拟。

## 17. 资金与持仓更新设计

### 17.1 买入更新流程

```text
成交买入
   ↓
计算成交金额
   ↓
计算佣金和滑点
   ↓
扣减现金
   ↓
增加总持仓数量
   ↓
增加今日买入数量
   ↓
更新成本价
   ↓
更新持仓市值
```

买入后：

```text
cash -= trade_amount + commission + slippage_cost
total_quantity += quantity
today_buy_quantity += quantity
```

### 17.2 卖出更新流程

```text
成交卖出
   ↓
计算成交金额
   ↓
计算佣金、印花税、滑点
   ↓
增加现金
   ↓
减少总持仓数量
   ↓
减少可卖数量
   ↓
计算已实现盈亏
   ↓
若持仓为 0，清空持仓记录
```

卖出后：

```text
cash += trade_amount - commission - tax - slippage_cost
total_quantity -= quantity
available_quantity -= quantity
```

### 17.3 每日持仓更新

每日收盘后：

```text
更新市场价格
更新持仓市值
更新浮动盈亏
更新持仓天数
更新最高价
更新总资产
更新回撤
```

### 17.4 T+1 可卖数量滚动

每个新交易日开始时：

```text
available_quantity += today_buy_quantity
today_buy_quantity = 0
```

## 18. 止损、止盈和清仓机制

### 18.1 固定止损

如果当前持仓收益率：

```text
position_return <= -stop_loss_ratio
```

生成卖出信号。

### 18.2 固定止盈

如果当前持仓收益率：

```text
position_return >= take_profit_ratio
```

生成卖出或减仓信号。

### 18.3 移动止盈

如果从持仓最高价回撤：

```text
current_price / highest_price_since_buy - 1 <= -trailing_stop_ratio
```

生成卖出信号。

### 18.4 最大持仓天数

如果：

```text
holding_days >= max_holding_days
```

生成卖出信号。

### 18.5 账户最大回撤清仓

如果账户回撤：

```text
account_drawdown <= -max_drawdown_clear_ratio
```

生成清仓信号。

### 18.6 清仓执行

清仓时：

```text
对所有可卖持仓生成卖出订单
不可卖部分保留
停牌或跌停无法卖出的持仓保留
```

## 21. 绩效指标设计

### 21.1 收益类指标

#### 总收益率

```text
total_return = final_value / initial_cash - 1
```

#### 年化收益率

```text
annual_return = (final_value / initial_cash) ^ (252 / trading_days) - 1
```

#### 超额收益

```text
excess_return = strategy_return - benchmark_return
```

### 21.2 风险类指标

#### 最大回撤

```text
drawdown_t = total_value_t / historical_max_value_t - 1
max_drawdown = min(drawdown_t)
```

#### 波动率

```text
annual_volatility = std(daily_returns) * sqrt(252)
```

### 21.3 风险调整收益指标

#### 夏普比率

```text
sharpe_ratio = (annual_return - risk_free_rate) / annual_volatility
```

#### 卡玛比率

```text
calmar_ratio = annual_return / abs(max_drawdown)
```

### 21.4 交易统计指标

```text
trade_count
win_rate
profit_loss_ratio
average_profit
average_loss
average_holding_days
turnover_rate
total_commission
total_tax
total_slippage
```

### 21.5 胜率

```text
win_rate = profitable_trades / total_closed_trades
```

### 21.6 盈亏比

```text
profit_loss_ratio = average_profit / abs(average_loss)
```

### 21.7 持仓周期

```text
average_holding_days = total_holding_days / closed_position_count
```

## 24. 防未来函数设计

### 24.1 未来函数风险点

常见未来函数风险包括：

```text
使用未来收盘价生成当前信号
使用未来最高价或最低价判断止盈止损
使用未来因子数据
使用未来成分股
使用未来停牌信息
使用未来成交结果修正当前信号
当前日收盘后信号却按当前日开盘成交
```

### 24.2 防未来函数机制

系统应采用：

```text
时间切片数据访问
上下文日期限制
信号日期和成交日期分离
默认 T+1 成交
指标 rolling 严格向后看
因子数据按 trade_date 截断
财务数据按披露日期处理，第一版可暂不做财务数据
```

### 24.3 默认安全规则

```text
T 日收盘后生成信号
T+1 日开盘执行
```

这是第一版推荐规则。

### 24.4 检查项

回测任务完成后，可执行未来函数检查：

```text
signal_date <= trade_date
如果使用 next_open，则 trade_date > signal_date
指标计算窗口不包含未来日期
因子计算窗口不包含未来收益
```

## 25. 回测和实盘一致性设计

### 25.1 一致性目标

回测和实盘不可能完全一致，但应尽量保证核心逻辑一致：


### 25.2 不一致来源

回测和实盘差异主要来自：

```text

停牌无法处理
实时行情延迟
券商拒单

网络异常
```

### 25.3 一致性设计措施

```text

统一风控规则配置
统一交易成本模型
统一 A 股规则模块
回测记录 signal_id
```

### 25.4 回测保守化

```text

```

### 31.5 结果一致性测试

相同输入重复执行回测，应得到一致结果。

## 32. 验收标准

### 32.1 基础功能验收

系统应满足：

```text
1. 能创建回测任务。
2. 能异步执行回测。
3. 能查询回测任务状态。
4. 能输出回测结果。
5. 能输出交易记录。
6. 能输出每日资产曲线。
7. 能处理回测失败。
```

### 32.2 策略回测验收

系统应满足：

```text
1. 能运行趋势跟踪策略。
2. 能运行均值回归策略。
3. 能运行因子打分策略。
4. 能正确生成买入信号。
5. 能正确生成卖出信号。
6. 能正确生成清仓信号。
```

### 32.3 A 股规则验收

系统应满足：

```text
1. 非交易日不推进交易。
2. 停牌日不交易。
3. 涨停时不买入。
4. 跌停时不卖出。
5. 当日买入不能当日卖出。
6. 买入数量为 100 股整数倍。
7. 卖出数量不超过可卖数量。
```

### 32.4 资金持仓验收

系统应满足：

```text
1. 买入后现金减少。
2. 卖出后现金增加。
3. 买入后持仓增加。
4. 卖出后持仓减少。
5. 手续费正确扣除。
6. 印花税正确扣除。
7. 滑点正确计算。
8. 总资产 = 现金 + 持仓市值。
```

### 32.5 绩效指标验收

系统应正确计算：

```text
总收益率
年化收益率
最大回撤
夏普比率
胜率
盈亏比
交易次数
换手率
手续费总额
最终资产
```

### 32.6 防未来函数验收

系统应满足：

```text
1. 默认 T 日信号 T+1 执行。
2. 策略不能访问未来行情。
3. 指标计算不能包含未来数据。
4. 因子计算不能使用未来收益作为特征。
```

## 36. 总结

本文档完成了 A 股低频量化策略研究、回测、模拟盘与实盘执行系统的回测引擎设计。

回测引擎的核心职责是：

```text
加载历史数据
按交易日推进
调用策略生成信号
将信号转为模拟订单
执行回测风控
模拟成交
更新现金和持仓
记录交易和资产曲线
计算绩效指标
生成回测报告
```

本设计重点保证：

```text
1. 回测服务于低频策略。
2. 回测不支持高频交易。
3. 默认采用 T 日信号、T+1 执行。
4. 明确模拟 A 股交易规则。
5. 明确模拟手续费、印花税和滑点。
6. 明确处理停牌、涨跌停和 T+1。
7. 明确记录现金、持仓、市值和总资产。
8. 明确输出收益、回撤、夏普、胜率等指标。
9. 明确防未来函数。
10. 明确与实盘逻辑尽量一致。
```

最终，回测引擎应帮助用户判断：

```text
策略在历史数据中是否有效？
收益是否来自真实可执行交易？
风险和回撤是否可接受？
交易成本是否会吞噬收益？
参数是否存在过拟合？
策略是否符合低频定位？
是否值得进入模拟盘验证？
```

回测通过不代表可以直接实盘。策略必须继续经过：

```text
样本外验证
参数稳定性检查
模拟盘运行
实盘风控测试
小资金灰度实盘
```

后，才允许进入正式实盘阶段。

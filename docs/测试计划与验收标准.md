# A 股低频量化策略研究、回测、模拟盘与实盘执行系统测试计划与验收标准

## 1. 引言

### 1.1 文档目的

本文档用于描述“A 股低频量化策略研究、回测、模拟盘与实盘执行系统”的测试计划与验收标准。该系统涉及历史行情数据、策略信号、回测引擎、参数优化、模拟盘、实盘交易接入、风控系统、订单管理、成交回报、账户同步和审计日志等多个模块，因此必须通过系统化测试保证功能正确性、数据一致性、实盘安全性和异常可控性。

本文档主要回答以下问题：

```text
1. 系统需要测试哪些内容？
2. 各模块如何进行单元测试、集成测试和系统测试？
3. 回测、模拟盘和实盘接口如何验证？
4. A 股交易规则如何测试？
5. 风控规则如何测试？
6. 实盘下单前需要通过哪些测试？
7. 哪些情况视为验收通过？
8. 哪些问题会导致项目不能进入实盘阶段？
```

---

## 2. 测试目标

### 2.1 总体目标

测试工作的总体目标是：

> 证明系统能够在低频量化交易场景下，稳定、正确、安全地完成数据管理、策略生成、回测分析、参数优化、模拟盘验证和实盘交易接入，并确保所有真实订单在提交券商前必须经过风控检查。

### 2.2 具体目标

系统测试需要达到以下目标：

```text
1. 验证数据导入、清洗、查询功能正确。
2. 验证指标和因子计算结果正确。
3. 验证策略信号生成逻辑正确。
4. 验证回测引擎能够正确模拟交易过程。
5. 验证手续费、印花税、滑点计算正确。
6. 验证 A 股 T+1、涨跌停、停牌、交易单位规则正确。
7. 验证参数优化流程可运行、可追溯。
8. 验证模拟盘账户、订单、成交和持仓逻辑正确。
9. 验证实盘账户资金和持仓同步正确。
10. 验证实盘下单、撤单、订单同步和成交回报正确。
11. 验证实盘风控规则有效。
12. 验证异常情况下系统默认不自动交易。
13. 验证一键停止机制有效。
14. 验证审计日志完整。
15. 验证系统不包含高频交易相关能力。
```

## 4. 测试策略

### 4.1 分层测试策略

系统测试分为以下层级：

```text
单元测试
模块测试
集成测试
系统测试
接口测试
回归测试
异常测试
实盘 Mock 测试
小资金灰度验收测试
```

### 4.2 测试阶段

建议按以下阶段执行：

```text
第一阶段：基础数据和数据库测试
第二阶段：指标、因子和策略测试
第三阶段：回测引擎测试
第四阶段：参数优化测试
第五阶段：模拟盘测试
第六阶段：风控测试
第七阶段：实盘接口 Mock 测试
第八阶段：系统集成测试
第九阶段：小资金灰度实盘测试
第十阶段：最终验收
```

### 4.3 测试优先级

测试优先级分为：

| 优先级 | 说明                    |
| --- | --------------------- |
| P0  | 实盘安全、资金持仓、订单风控相关，必须通过 |
| P1  | 核心业务流程，必须通过           |
| P2  | 报告、统计、查询类功能，基本通过      |
| P3  | 辅助功能，可后续优化            |

P0 级测试失败时，系统不得进入实盘阶段。

## 26. 缺陷等级定义

| 等级       | 说明          | 示例             |
| -------- | ----------- | -------------- |
| Blocker  | 阻断系统运行或实盘安全 | 风控失效、重复下单      |
| Critical | 严重影响核心功能    | 订单状态错误、成交重复    |
| Major    | 影响主要功能      | 回测结果错误、模拟盘资金错误 |
| Minor    | 一般问题        | 报告字段缺失         |
| Trivial  | 轻微问题        | 文案、格式问题        |

### 26.1 缺陷处理要求

```text
Blocker：必须立即修复，未修复不得继续测试或实盘
Critical：必须在验收前修复
Major：原则上验收前修复
Minor：可记录后续修复
Trivial：可后续优化
```

## 29. 总结

本文档完成了 A 股低频量化策略研究、回测、模拟盘与实盘执行系统的测试计划与验收标准设计。

测试工作的核心目标是：

```text
证明系统功能可用；
证明交易逻辑正确；
证明资金和持仓计算正确；
证明 A 股规则生效；
证明风控不能被绕过；
证明异常状态下不会继续自动交易；
证明每一笔实盘交易都可追溯；
证明系统长期符合低频交易定位。
```

本系统最终验收的核心标准是：

```text
能回测，但不迷信回测；
能模拟盘，但不跳过风控；
能接实盘，但默认关闭实盘；
能下单，但风控不通过绝不下单；
能恢复异常，但状态不确定时绝不自动交易；
能运行策略，但绝不演化为高频交易系统。
```

只有在以下条件全部满足后，系统才具备进入小资金灰度实盘的基础：

```text
回测通过；
样本外验证通过；
模拟盘稳定；
Mock 券商测试通过；
风控测试通过；
一键停止测试通过；
重复下单防护通过；
unknown 订单恢复通过；
审计日志完整；
人工确认风险可控。
```

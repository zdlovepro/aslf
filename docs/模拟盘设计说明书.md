# A 股低频量化策略研究、回测、模拟盘与实盘执行系统模拟盘设计说明书

## 1. 引言

### 1.1 文档目的

本文档用于描述“A 股低频量化策略研究、回测、模拟盘与实盘执行系统”的模拟盘设计方案。模拟盘是连接历史回测和真实实盘之间的验证环境，用于在不使用真实资金的情况下，模拟策略在真实交易日、真实或准实时行情、真实交易规则约束下的运行效果。

本文档主要回答以下问题：

```text
1. 模拟盘在整个系统中承担什么作用？
2. 模拟盘和回测、实盘有什么区别？
3. 模拟盘账户如何设计？
4. 模拟盘订单和成交如何设计？
5. 模拟盘如何接收策略信号？
6. 模拟盘如何生成交易计划？
7. 模拟盘如何执行风控？
8. 模拟盘如何模拟成交？
9. 模拟盘如何维护资金和持仓？
10. 模拟盘如何输出绩效报告？
11. 模拟盘如何为后续实盘提供验证依据？
```

## 2. 模拟盘定位

### 2.1 系统定位

模拟盘是策略进入实盘前的必要验证环节。

完整策略上线链路为：

```text
策略设计
   ↓
历史回测
   ↓
参数优化
   ↓
样本外验证
   ↓
模拟盘运行
   ↓
Mock 券商接口测试
   ↓
小资金灰度实盘
   ↓
正式实盘
```

模拟盘的定位是：

> 使用虚拟资金和模拟账户，在真实交易日和真实行情环境下验证低频策略的稳定性、交易计划生成能力、订单执行逻辑、风控规则、资金持仓更新和报告统计能力。

### 2.2 模拟盘与回测的区别

| 对比项  | 回测         | 模拟盘        |
| ---- | ---------- | ---------- |
| 数据来源 | 历史行情       | 实时或准实时行情   |
| 时间推进 | 一次性跑完整历史区间 | 按真实交易日逐步运行 |
| 账户资金 | 历史模拟账户     | 当前模拟账户     |
| 成交方式 | 历史价格模拟成交   | 当前行情模拟成交   |
| 策略运行 | 离线批量运行     | 定时或手动触发运行  |
| 订单状态 | 简化模拟       | 更接近实盘订单状态  |
| 风控   | 回测风控       | 接近实盘风控     |
| 目的   | 验证历史表现     | 验证接近实盘的稳定性 |

### 2.3 模拟盘与实盘的区别

| 对比项 | 模拟盘     | 实盘      |
| --- | ------- | ------- |
| 资金  | 虚拟资金    | 真实资金    |
| 账户  | 模拟账户    | 券商账户    |
| 下单  | 模拟订单    | 券商真实委托  |
| 成交  | 按规则模拟成交 | 券商真实成交  |
| 撤单  | 模拟撤单    | 券商撤单    |
| 风控  | 接近实盘风控  | 严格实盘风控  |
| 风险  | 无真实资金损失 | 有真实资金风险 |
| 作用  | 实盘前验证   | 真实交易执行  |

### 2.4 模拟盘不支持范围

模拟盘长期不支持以下能力：

```text
高频交易模拟
Tick 级成交模拟
Level-2 盘口模拟
逐笔成交回放
订单簿排队模拟
做市策略模拟
盘口套利模拟
高频撤单模拟
毫秒级低延迟仿真
```

模拟盘只服务于低频策略验证。

## 3. 模拟盘设计目标

### 3.1 总体目标

模拟盘的总体目标是：

> 在不使用真实资金的情况下，尽可能接近实盘流程，验证策略从信号生成、交易计划生成、风控检查、模拟下单、模拟成交、资金持仓更新到绩效报告输出的完整链路。

### 3.2 具体目标

模拟盘系统需要实现：

```text
1. 支持创建模拟账户。
2. 支持设置模拟账户初始资金。
3. 支持绑定策略实例。
4. 支持按交易日运行策略。
5. 支持读取实时或准实时行情。
6. 支持生成模拟交易计划。
7. 支持执行模拟盘风控。
8. 支持生成模拟订单。
9. 支持模拟成交。
10. 支持模拟撤单。
11. 支持维护模拟资金。
12. 支持维护模拟持仓。
13. 支持记录模拟成交。
14. 支持输出模拟盘资产曲线。
15. 支持输出模拟盘绩效报告。
16. 支持记录策略信号、风控结果和订单日志。
17. 支持判断策略是否满足进入实盘候选条件。
```

## 4. 模拟盘设计原则

### 4.1 低频交易原则

模拟盘只验证低频策略。

允许：

```text
日线策略
周线策略
月线策略
低频分钟级辅助执行
收盘后生成交易计划
次日或盘中低频执行
```

不允许：

```text
Tick 级策略
秒级高频信号
盘口套利
做市报价
高频撤单
毫秒级交易模拟
```

### 4.2 接近实盘原则

模拟盘应尽量复用实盘流程：

```text
策略信号
   ↓
交易计划
   ↓
风控检查
   ↓
订单管理
   ↓
成交处理
   ↓
资金持仓更新
   ↓
报告统计
```

区别只是模拟盘不调用真实券商接口。

### 4.3 策略不直接操作账户原则

策略只能输出：

```text
买入信号
卖出信号
减仓信号
清仓信号
目标仓位
信号原因
```

策略不得直接修改模拟账户资金和持仓。

### 4.4 风控前置原则

模拟订单生成前必须经过模拟盘风控。

风控失败时：

```text
不生成成交
记录风控事件
记录拒绝原因
```

### 4.5 结果可追溯原则

模拟盘中每一个订单和成交都应能追溯到：

```text
策略实例
策略参数版本
策略信号
交易计划
风控结果
模拟订单
模拟成交
账户资金变化
持仓变化
```

### 4.6 与实盘隔离原则

模拟盘数据必须与实盘数据隔离。

模拟盘不得：

```text
调用券商下单接口
修改实盘账户
读取实盘账户作为成交依据
将模拟订单误认为实盘订单
```

## 6. 模拟盘账户设计

### 6.1 模拟账户职责

模拟账户用于维护虚拟资金和资产状态。

主要职责：

```text
保存初始资金
维护当前现金
维护冻结资金
维护持仓市值
维护账户总资产
计算每日收益
计算最大回撤
记录账户状态
```

### 6.2 模拟账户字段

```text
simulation_account_id
account_name
initial_cash
current_cash
frozen_cash
market_value
total_asset
available_cash
daily_pnl
total_pnl
max_total_asset
drawdown
status
created_at
updated_at
```

### 6.3 模拟账户状态

```text
active       正常运行
paused       暂停运行
closed       已关闭
error        异常状态
```

### 6.4 创建模拟账户

创建模拟账户时需要：

```text
账户名称
初始资金
默认风控配置
绑定策略实例，可选
```

示例：

```json
{
  "name": "trend_strategy_sim_account",
  "initial_cash": 1000000,
  "risk_config": {
    "max_total_position": 0.8,
    "max_position_per_stock": 0.1,
    "min_cash_ratio": 0.2
  }
}
```

### 6.5 账户资产计算

```text
total_asset = current_cash + market_value
```

其中：

```text
market_value = Σ(position_quantity * latest_price)
```

### 6.6 当前回撤计算

```text
drawdown = (total_asset - max_total_asset) / max_total_asset
```

## 7. 模拟持仓设计

### 7.1 模拟持仓职责

模拟持仓模块负责维护每只股票的持仓数量、可卖数量、成本价、市值和盈亏。

### 7.2 模拟持仓字段

```text
simulation_position_id
simulation_account_id
symbol
total_quantity
available_quantity
today_buy_quantity
frozen_quantity
cost_price
market_price
market_value
unrealized_pnl
realized_pnl
holding_days
highest_price_since_buy
last_buy_time
last_sell_time
updated_at
```

### 7.3 A 股 T+1 可卖设计

模拟盘必须模拟 A 股 T+1。

买入当日：

```text
total_quantity 增加
today_buy_quantity 增加
available_quantity 不增加
```

下一交易日：

```text
available_quantity += yesterday_today_buy_quantity
today_buy_quantity = 0
```

卖出时：

```text
sell_quantity <= available_quantity
```

### 7.4 持仓成本价

买入后采用移动加权平均成本：

```text
new_cost_price =
(old_cost_price * old_quantity + buy_price * buy_quantity + buy_cost)
/
(old_quantity + buy_quantity)
```

### 7.5 持仓市值

```text
market_value = total_quantity * latest_price
```

### 7.6 浮动盈亏

```text
unrealized_pnl = market_value - cost_price * total_quantity
```

### 7.7 持仓天数

只要持仓数量大于 0，每经过一个交易日：

```text
holding_days += 1
```

清仓后：

```text
holding_days = 0
highest_price_since_buy = null
```

```text

   ↓

   ↓

   ↓
```

```text

```

## 9. 模拟盘调度设计

### 9.1 调度职责

模拟盘调度模块负责在合适的时间触发策略运行。

职责包括：

```text
检查交易日
检查策略状态
检查模拟账户状态
触发策略信号生成
触发交易计划生成
触发模拟执行
触发账户更新
触发报告更新
```

### 9.2 调度方式

支持以下方式：

```text
手动触发
定时触发
交易日收盘后触发
交易时段内低频触发
```

### 9.3 日线策略推荐调度

推荐流程：

```text
15:30 后更新行情
   ↓
计算指标和因子
   ↓
策略生成信号
   ↓
生成次日模拟交易计划
   ↓
次日交易时段模拟执行
```

### 9.4 盘中低频策略调度

对于低频分钟级辅助策略：

```text
每 30 分钟或 60 分钟执行一次
```

不允许秒级循环触发。

### 9.5 调度前检查

每次调度前必须检查：

```text
当前是否交易日
策略是否 running
账户是否 active
是否触发一键停止
行情是否可用
是否超过策略频率限制
```

## 10. 模拟盘运行流程

### 10.1 总体流程

```text
启动模拟盘
   ↓
检查账户和策略状态
   ↓
读取交易日历
   ↓
读取行情数据
   ↓
计算指标和因子
   ↓
策略生成信号
   ↓
生成交易计划
   ↓
模拟盘风控检查
   ↓
生成模拟订单
   ↓
模拟撮合成交
   ↓
更新账户和持仓
   ↓
记录订单和成交
   ↓
更新资产曲线
   ↓
生成报告
```

### 10.2 日线模拟盘流程

```text
T 日收盘后：
    更新 T 日行情
    计算 T 日指标和因子
    策略生成信号
    生成 T+1 交易计划

T+1 交易时段：
    获取最新行情
    执行模拟风控
    生成模拟订单
    按模拟成交规则成交
    更新模拟账户和持仓
```

### 10.3 手动运行流程

```text
用户点击运行模拟盘
   ↓
选择策略实例
   ↓
选择模拟账户
   ↓
指定运行日期
   ↓
系统执行一次完整模拟流程
```

## 12. 模拟盘风控设计

### 12.1 风控目标

模拟盘风控用于验证策略在接近实盘条件下是否可执行。

### 12.2 风控检查项

```text
模拟账户资金检查
模拟持仓可卖数量检查
总仓位限制
单票仓位限制
最低现金比例
停牌检查
涨跌停检查
T+1 检查
交易日检查
交易时间检查
交易频率检查
重复订单检查
单日亏损检查
最大回撤检查
```

### 12.3 风控流程

```text
候选模拟订单
   ↓
检查交易日
   ↓
检查交易时间
   ↓
检查股票状态
   ↓
检查账户资金
   ↓
检查持仓可卖数量
   ↓
检查仓位限制
   ↓
检查频率限制
   ↓
通过 / 拒绝
```

### 12.4 风控拒绝处理

如果风控拒绝：

```text
不生成模拟成交
模拟订单状态为 rejected
记录拒绝原因
记录风控事件
更新模拟盘报告
```

### 12.5 风控事件类型

```text

limit_down_cannot_sell
t_plus_one_restricted

max_total_position_exceeded

frequency_violation
drawdown_limit_triggered
```

### 14.3 撮合输出

```text
成交价格
成交数量
手续费
印花税
滑点成本
成交时间
拒绝原因
```

### 14.4 成交模式

第一版建议支持：

```text
full_fill_if_tradable
```

即：

```text
如果股票可交易，则全部成交；
如果股票不可交易，则拒绝成交。
```

后续可扩展：

```text
partial_fill
volume_limited_fill
bid_ask_fill
```

### 14.5 成交价格模式

支持以下模式：

| 模式                 | 说明            |
| ------------------ | ------------- |
| latest_price       | 使用最新价成交       |
| open_price         | 使用开盘价成交       |
| close_price        | 使用收盘价成交       |
| bid_ask_price      | 使用买一/卖一价成交    |
| limit_price        | 使用委托价格成交      |
| conservative_price | 使用对策略更保守的价格成交 |

第一版推荐：

```text
latest_price + slippage
```

或：

```text
next_open + slippage
```

### 14.6 买入滑点

```text
actual_buy_price = base_price * (1 + slippage_rate)
```

### 14.7 卖出滑点

```text
actual_sell_price = base_price * (1 - slippage_rate)
```

### 14.8 涨停买入规则

如果股票涨停：

```text
allow_buy_limit_up = false
```

则买入订单拒绝。

### 14.9 跌停卖出规则

如果股票跌停：

```text
allow_sell_limit_down = false
```

则卖出订单拒绝。

### 14.10 停牌规则

停牌股票：

```text
不允许买入
不允许卖出
已有持仓继续保留
估值使用最近可用价格
```

### 15.2 成交金额

```text

```

### 15.3 买入成本

```text

tax = 0

```

```text

```

## 16. 模拟账户资金更新设计

### 16.1 买入资金更新

买入成交后：

```text
current_cash =
current_cash - amount - commission - slippage_cost
```

同时：

```text
持仓数量增加
今日买入数量增加
可卖数量不增加
```

### 16.2 卖出资金更新

卖出成交后：

```text
current_cash =
current_cash + amount - commission - tax - slippage_cost
```

同时：

```text
持仓数量减少
可卖数量减少
```

### 16.3 冻结资金设计

第一版模拟盘可以简化冻结资金。

如果要模拟更接近实盘，可在订单 submitted 后冻结资金或持仓：

买入冻结：

```text
frozen_cash += estimated_amount + estimated_cost
available_cash -= estimated_amount + estimated_cost
```

卖出冻结：

```text
frozen_quantity += quantity
available_quantity -= quantity
```

成交或撤单后释放冻结。

第一版建议：

```text
可交易即成交，不复杂模拟冻结；
后续再扩展冻结资金和冻结持仓。
```

## 17. 模拟持仓更新设计

### 17.1 买入后持仓更新

```text

available_quantity 不变
cost_price 更新为加权平均成本
market_price 更新为最新价
```

### 17.2 卖出后持仓更新

```text

```

如果 total_quantity = 0：

```text

```

### 17.3 每日开始时 T+1 滚动

```text
available_quantity += yesterday_today_buy_quantity
today_buy_quantity = 0
```

### 17.4 每日收盘持仓估值

每日收盘或每次行情更新后：

```text
market_price = latest_price
market_value = total_quantity * market_price
unrealized_pnl = market_value - cost_price * total_quantity
```

## 23. 模拟盘与回测一致性设计

### 23.1 一致性目标

模拟盘应尽量复用回测模块中的以下逻辑：

```text
策略信号生成
指标计算
因子计算
交易成本计算
A 股交易规则
仓位管理
止损止盈规则
绩效指标计算
```

### 23.2 差异点

| 项目 | 回测      | 模拟盘      |
| -- | ------- | -------- |
| 时间 | 历史时间    | 当前真实时间   |
| 行情 | 历史行情    | 实时或准实时行情 |
| 成交 | 历史价格模拟  | 当前行情模拟   |
| 执行 | 一次性批量执行 | 按日或定时执行  |
| 状态 | 任务完成即结束 | 长期运行     |

### 23.3 统一配置

建议回测和模拟盘使用相同的：

```text
策略参数
仓位参数
成本参数
风控参数
A 股规则参数
```

这样才能比较回测表现和模拟盘表现是否一致。

## 24. 模拟盘与实盘一致性设计

### 24.1 一致性目标

模拟盘应尽量模拟实盘流程：

```text

模拟不会出现券商拒单

模拟不会出现真实部分成交，除非主动设计
模拟不会产生真实资金损失
```

```text

模拟涨跌停不能成交
模拟停牌不能交易
模拟 T+1 可卖规则
```

```text
1. 模拟盘连续运行稳定。
2. 无严重系统异常。
3. 无异常高频信号。
4. 无重复订单。
5. 风控拒单原因合理。
6. 最大回撤可接受。
7. 换手率符合低频定位。
8. 策略表现与回测结果没有严重背离。
9. 交易成本未明显吞噬收益。
10. 模拟盘报告完整。
```

### 27.4 不允许进入实盘的情况

出现以下情况，不应进入实盘：

```text
模拟盘频繁报错
策略频繁触发风控
信号频率异常
订单数量异常
资金或持仓计算错误
最大回撤超过阈值
模拟盘表现和回测严重不一致
交易成本过高
策略解释不清楚
```

## 29. 模拟盘验收标准

### 29.1 账户验收

系统应满足：

```text
1. 能创建模拟账户。
2. 能查询模拟账户。
3. 能维护当前现金。
4. 能维护持仓市值。
5. 能计算账户总资产。
6. 能记录每日资产。
```

### 29.2 策略验收

系统应满足：

```text
1. 能绑定策略实例。
2. 能启动模拟盘策略。
3. 能暂停模拟盘策略。
4. 能停止模拟盘策略。
5. 能按交易日生成策略信号。
6. 能记录策略信号。
```

### 29.3 订单成交验收

系统应满足：

```text
1. 能生成模拟交易计划。
2. 能生成模拟订单。
3. 能执行模拟风控。
4. 风控拒绝时不成交。
5. 风控通过时能模拟成交。
6. 能记录模拟成交。
7. 能更新订单状态。
8. 能查询订单和成交。
```

### 29.4 持仓资金验收

系统应满足：

```text
1. 买入后现金减少。
2. 买入后持仓增加。
3. 当日买入不可当日卖出。
4. 下一交易日可卖数量增加。
5. 卖出后现金增加。
6. 卖出后持仓减少。
7. 手续费和印花税正确扣除。
8. 总资产计算正确。
```

### 29.5 风控验收

系统应满足：

```text
1. 资金不足时拒绝买入。
2. 可卖数量不足时拒绝卖出。
3. 停牌股票拒绝交易。
4. 涨停买入被拒绝。
5. 跌停卖出被拒绝。
6. 超过仓位限制时拒绝订单。
7. 超过频率限制时暂停策略。
8. 风控事件能被记录。
```

### 29.6 报告验收

系统应满足：

```text
1. 能生成模拟盘报告。
2. 能生成资产曲线。
3. 能统计交易次数。
4. 能统计胜率。
5. 能统计最大回撤。
6. 能统计总交易成本。
7. 能统计风控拒单记录。
8. 能统计策略交易频率。
```

## 32. 与其他模块的关系

### 32.1 与数据模块关系

模拟盘依赖数据模块提供：

```text
股票基础信息
交易日历
日线行情
分钟行情，可选
停牌状态
涨跌停价格
```

### 32.2 与行情模块关系

模拟盘依赖行情模块提供：

```text
最新价
开盘价
收盘价
涨停价
跌停价
停牌状态
行情时间戳
```

### 32.3 与策略模块关系

模拟盘调用策略模块生成信号。

策略不直接修改模拟账户。

### 32.4 与仓位模块关系

仓位模块将目标仓位转换为模拟订单数量。

### 32.5 与风控模块关系

模拟盘复用接近实盘的风控逻辑。

### 32.6 与报告模块关系

报告模块读取模拟账户、订单、成交、持仓和资产曲线数据，生成模拟盘报告。

### 32.7 与实盘模块关系

模拟盘通过后，策略才能进入实盘候选。

但模拟盘不调用券商接口，不产生真实订单。

## 33. 总结

本文档完成了 A 股低频量化策略研究、回测、模拟盘与实盘执行系统的模拟盘设计。

模拟盘的核心职责是：

```text
使用虚拟账户验证策略运行稳定性；
使用真实或准实时行情验证交易计划；
使用模拟订单和成交验证执行逻辑；
使用接近实盘的风控验证安全性；
使用资产曲线和报告验证策略表现；
为后续实盘接入提供依据。
```

模拟盘设计的核心原则是：

```text
策略只生成信号；
模拟盘不直接接触真实资金；
模拟盘不调用真实券商下单接口；
模拟盘必须执行风控；
模拟盘必须记录订单、成交和持仓；
模拟盘必须统计收益、回撤和交易频率；
模拟盘必须服务低频交易定位；
模拟盘通过不代表可以直接正式实盘。
```

最终，模拟盘应帮助用户判断：

```text
策略是否能稳定运行？
策略信号是否合理？
交易计划是否可执行？
仓位和资金计算是否正确？
风控规则是否有效？
交易成本是否可接受？
策略频率是否符合低频定位？
模拟盘表现是否与回测结果基本一致？
是否具备进入小资金实盘灰度的条件？
```

模拟盘是从“历史验证”走向“真实资金执行”的过渡阶段。只有模拟盘稳定、风控有效、报告完整、异常可控，策略才应进入实盘候选阶段。

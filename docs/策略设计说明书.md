# A 股低频量化策略研究、回测、模拟盘与实盘执行系统策略设计说明书

## 1. 引言

### 1.1 文档目的

本文档用于描述“A 股低频量化策略研究、回测、模拟盘与实盘执行系统”的策略设计方案。本文档在软件需求规格说明书、系统概要设计说明书和数据库设计说明书的基础上，进一步细化系统中的低频交易策略体系、策略运行流程、信号生成机制、参数空间、仓位设计、风控联动、回测与实盘一致性、防未来函数机制、防过拟合机制和策略频率限制。

本文档主要回答以下问题：

1. 系统支持哪些低频策略？
2. 策略如何生成买入、卖出、减仓和清仓信号？
3. 策略需要训练或优化哪些参数？
4. 策略如何决定买多少、卖多少、留多少现金？
5. 策略如何与仓位管理和风控模块协作？
6. 策略如何保证回测、模拟盘和实盘逻辑一致？
7. 策略如何避免未来函数和过拟合？
8. 策略如何保证长期不演化成高频交易？

### 1.2 适用范围

本文档适用于以下策略类型：

* 趋势跟踪策略
* 均值回归策略
* 因子打分选股策略
* 组合型低频策略
* 机器学习预测策略预留

本文档不适用于以下策略类型：

* 高频交易策略
* Tick 级策略
* Level-2 盘口策略
* 逐笔成交策略
* 做市策略
* 盘口套利策略
* 高频撤单策略
* 毫秒级或微秒级低延迟策略

### 1.3 策略设计目标

策略设计目标如下：

1. 构建可解释、可回测、可优化、可实盘执行的低频策略体系。
2. 支持通过参数优化寻找较优策略参数。
3. 支持基于技术指标、因子评分和风险状态生成交易信号。
4. 支持输出目标仓位，而不是直接下单。
5. 支持回测、模拟盘和实盘共用统一策略逻辑。
6. 支持策略频率限制，防止异常高频触发。
7. 支持防未来函数和防过拟合设计。
8. 支持策略信号、参数版本和交易原因可追溯。

### 1.4 策略设计原则

#### 1.4.1 低频原则

系统所有策略均为低频策略，主要基于日线数据运行，可选使用分钟级数据进行辅助确认。策略不得依赖 Tick 级行情、Level-2 盘口、逐笔成交或低延迟交易环境。

#### 1.4.2 信号与执行分离原则

策略模块只负责生成信号和目标仓位，不直接调用订单接口，不直接修改账户资金和持仓。

策略输出：

```text
买入信号
卖出信号
减仓信号
清仓信号
目标仓位
交易原因
信号强度
```

订单执行由交易计划、仓位管理、风控模块和订单管理模块完成。

#### 1.4.3 可解释原则

策略生成的每个信号都应包含原因说明，例如：

```text
MA20 上穿 MA60
RSI 低于 30
因子评分进入前 10%
触发 8% 止损
账户回撤超过阈值
```

#### 1.4.4 可配置原则

策略核心阈值和规则不得硬编码，应通过参数配置或参数优化产生。

#### 1.4.5 可追溯原则

每个策略信号必须记录：

```text
策略 ID
策略实例 ID
参数版本 ID
股票代码
信号日期
信号类型
信号强度
目标仓位
信号原因
生成时间
```

#### 1.4.6 风控优先原则

策略信号不等于最终订单。策略信号必须经过仓位管理和风控模块检查后，才可能进入模拟盘或实盘订单流程。

#### 1.4.7 防未来函数原则

策略在任意时点只能使用当前时点及之前已经可获得的数据，不得使用未来行情、未来财务数据或未来成分股信息。

#### 1.4.8 防过拟合原则

策略参数优化必须进行样本内训练和样本外验证，不能只追求历史最高收益率。

## 2. 策略体系总体设计

### 2.1 策略分类

系统第一阶段支持三类基础策略：

| 策略类型   | 说明                  | 适用场景        |
| ------ | ------------------- | ----------- |
| 趋势跟踪策略 | 捕捉上涨趋势，顺势买入，趋势破坏后卖出 | 趋势行情、强势股行情  |
| 均值回归策略 | 捕捉短期超跌后的反弹机会        | 震荡行情、超跌反弹行情 |
| 因子打分策略 | 根据多因子评分选择股票组合       | 中长期选股、组合调仓  |

后续可扩展：

| 扩展策略     | 说明                |
| -------- | ----------------- |
| 组合型策略    | 将趋势、均值回归和因子评分组合使用 |
| 机器学习预测策略 | 使用模型预测未来收益或上涨概率   |
| 风险平价仓位策略 | 根据波动率调整仓位         |
| 指数择时策略   | 根据大盘状态控制总仓位       |

### 2.2 策略生命周期

策略生命周期如下：

```text
策略定义
   ↓
策略参数配置
   ↓
策略回测
   ↓
参数优化
   ↓
样本外验证
   ↓
模拟盘运行
   ↓
实盘候选
   ↓
小资金实盘
   ↓
正式实盘运行
   ↓
策略监控与复盘
```

### 2.3 策略运行模式

策略支持三种运行模式：

| 模式         | 说明                         |
| ---------- | -------------------------- |
| backtest   | 历史回测模式，使用历史行情模拟交易          |
| simulation | 模拟盘模式，使用实时或准实时行情模拟运行       |
| live       | 实盘模式，生成真实交易计划，但仍需风控和订单模块执行 |

三种模式应尽可能共用同一套策略逻辑。

### 2.4 策略基础接口设计

所有策略应继承统一策略基类。

```text
BaseStrategy
├── initialize(params)
├── validate_params(params)
├── prepare_data(context)
├── generate_signals(context)
├── generate_target_position(context)
├── explain_signal(signal)
└── get_required_indicators()
```

#### initialize(params)

用于初始化策略参数。

#### validate_params(params)

用于校验参数是否合法，例如：

* 短均线周期必须小于长均线周期。
* 止损比例必须大于 0。
* 最大仓位不能超过 1。
* 策略频率必须符合低频限制。

#### prepare_data(context)

用于读取策略所需行情、指标、因子和持仓数据。

#### generate_signals(context)

用于生成买入、卖出、减仓、清仓和持有信号。

#### generate_target_position(context)

用于根据策略信号生成目标仓位。

#### explain_signal(signal)

用于生成信号原因说明。

#### get_required_indicators()

用于声明策略需要的指标，例如：

```text
MA20
MA60
MACD
RSI
BOLL
ATR
```

## 3. 策略上下文设计

### 3.1 策略上下文说明

策略上下文是策略运行时可访问的信息集合。策略只能通过上下文读取数据，不得直接访问数据库和订单接口。

### 3.2 上下文字段

策略上下文应包含：

```text
current_date              当前日期
mode                      运行模式：backtest / simulation / live
symbols                   股票池
market_data               行情数据
indicator_data            指标数据
factor_data               因子数据
benchmark_data            基准指数数据
current_positions         当前持仓
current_cash              当前现金
account_value             当前账户总资产
risk_state                当前风险状态
params                    策略参数
calendar                  交易日历
```

### 3.3 不允许访问的数据

策略不得访问：

1. 当前日期之后的行情数据。
2. 当前日期之后的指标数据。
3. 当前日期之后的因子数据。
4. 未来成交结果。
5. 未来资金和持仓状态。
6. 未公开或不可获得的未来财务数据。

## 4. 策略信号设计

### 4.1 信号类型

系统支持以下信号类型：

| 信号类型   | 说明           |
| ------ | ------------ |
| buy    | 买入信号         |
| sell   | 卖出信号         |
| reduce | 减仓信号         |
| clear  | 清仓信号         |
| hold   | 持有信号         |
| watch  | 观察信号，不产生交易计划 |

### 4.2 信号字段设计

策略信号应包含以下字段：

```text
signal_id
strategy_id
strategy_instance_id
params_id
symbol
signal_date
signal_time
signal_type
signal_strength
target_position
confidence
reason
source_mode
created_at
```

### 4.3 信号强度设计

signal_strength 用于表示信号强弱，范围建议为：

```text
0.0 ~ 1.0
```

示例：

| strength  | 含义   |
| --------- | ---- |
| 0.0 - 0.3 | 弱信号  |
| 0.3 - 0.6 | 中等信号 |
| 0.6 - 0.8 | 强信号  |
| 0.8 - 1.0 | 极强信号 |

信号强度可参与仓位计算，但不得绕过风控限制。

### 4.4 目标仓位设计

target_position 表示策略建议的目标仓位比例，例如：

```text
0.00 表示不持仓
0.05 表示 5% 账户资金
0.10 表示 10% 账户资金
0.20 表示 20% 账户资金
```

策略只能建议目标仓位，最终能否执行由仓位管理和风控模块决定。

### 4.5 信号去重设计

为了防止重复信号，系统应在同一策略实例、同一股票、同一日期、同一信号类型上进行去重。

去重规则：

```text
strategy_instance_id + symbol + signal_date + signal_type
```

对于实盘模式，还应增加短时间重复信号过滤。

## 6. 趋势跟踪策略设计

### 6.1 策略思想

趋势跟踪策略的核心思想是：

> 当股票价格形成明显上升趋势时买入，趋势破坏或风险触发时卖出。

该策略适合趋势较明显的市场环境，通常用于捕捉强势股或中期上涨行情。

### 6.2 输入数据

趋势跟踪策略需要以下数据：

```text
收盘价
成交量
MA5 / MA10 / MA20 / MA60 / MA120
MACD
成交量均线
ATR
大盘指数趋势
当前持仓
账户仓位
```

### 6.3 买入条件设计

基础买入条件：

```text
close > MA_short
MA_short > MA_long
MACD > buy_macd_threshold
volume > volume_ma20 * buy_volume_multiplier
market_filter_passed = true
```

示例：

```text
收盘价 > MA20
MA20 > MA60
MACD > 0
成交量 > 20 日均量 * 1.2
沪深 300 收盘价 > 沪深 300 MA60
```

### 6.4 卖出条件设计

基础卖出条件：

```text
close < sell_ma
MACD < sell_macd_threshold
holding_days > max_holding_days
trigger_stop_loss = true
trigger_trailing_stop = true
```

### 6.5 信号强度计算

趋势策略信号强度可由以下因素组成：

```text
trend_score = w1 * ma_trend_score
            + w2 * macd_score
            + w3 * volume_score
            + w4 * market_score
            - w5 * volatility_penalty
```

其中：

* ma_trend_score：均线多头排列程度。
* macd_score：MACD 强度。
* volume_score：成交量放大程度。
* market_score：大盘趋势状态。
* volatility_penalty：波动率惩罚。

### 6.6 目标仓位设计

趋势信号的目标仓位可按信号强度调整：

```text
base_position = initial_position_ratio
adjusted_position = base_position * signal_strength_multiplier
final_position = min(adjusted_position, max_position_per_stock)
```

示例：

| 信号强度      | 目标仓位 |
| --------- | ---- |
| 0.3 - 0.5 | 5%   |
| 0.5 - 0.7 | 10%  |
| 0.7 - 0.9 | 15%  |
| 0.9 - 1.0 | 20%  |

最终仓位仍必须经过仓位管理模块和风控模块限制。

### 6.7 趋势策略参数空间

| 参数                     | 建议搜索范围                   |
| ---------------------- | ------------------------ |
| ma_short               | [5, 10, 20]              |
| ma_long                | [30, 60, 120]            |
| buy_volume_multiplier  | [1.0, 1.2, 1.5, 2.0]     |
| buy_macd_threshold     | [-0.2, 0, 0.2]           |
| stop_loss_ratio        | [0.05, 0.08, 0.10, 0.12] |
| take_profit_ratio      | [0.10, 0.20, 0.30, 0.50] |
| trailing_stop_ratio    | [0.05, 0.08, 0.10, 0.15] |
| initial_position_ratio | [0.05, 0.10, 0.15]       |
| max_position_per_stock | [0.10, 0.15, 0.20]       |

### 6.8 趋势策略伪代码

```text
for each trade_date:
    for each symbol in stock_pool:
        data = get_history_until(trade_date, symbol)
        indicators = get_indicators_until(trade_date, symbol)

        if not holding(symbol):
            if close > ma_short and ma_short > ma_long and macd > threshold:
                if volume > volume_ma20 * volume_multiplier:
                    if market_filter_passed:
                        generate buy signal
        else:
            if close < sell_ma:
                generate sell signal
            elif stop_loss_triggered:
                generate sell signal
            elif trailing_stop_triggered:
                generate sell signal
```

## 7. 均值回归策略设计

### 7.1 策略思想

均值回归策略的核心思想是：

> 股票价格短期大幅偏离均值后，未来有一定概率向均值回归。

该策略适合震荡行情和短期超跌反弹场景。

### 7.2 输入数据

均值回归策略需要以下数据：

```text
收盘价
RSI
BOLL 上中下轨
MA20
过去 N 日收益率
成交量
ATR
当前持仓
```

### 7.3 买入条件设计

基础买入条件：

```text
RSI < buy_rsi_threshold
close < boll_lower
return_Nd < decline_threshold
is_not_limit_down
is_not_suspended
```

示例：

```text
RSI14 < 30
收盘价 < BOLL 下轨
过去 5 日跌幅 < -8%
股票未停牌
股票未连续跌停
```

### 7.4 卖出条件设计

基础卖出条件：

```text
RSI > sell_rsi_threshold
close >= MA20
close >= boll_mid
trigger_take_profit
trigger_stop_loss
holding_days > max_holding_days
```

### 7.5 信号强度计算

均值回归信号强度可由以下因素组成：

```text
reversion_score = w1 * rsi_oversold_score
                + w2 * boll_deviation_score
                + w3 * decline_score
                - w4 * volatility_penalty
                - w5 * liquidity_penalty
```

### 7.6 目标仓位设计

均值回归策略通常风险较高，不建议单次仓位过大。

建议：

```text
initial_position_ratio = 0.05 - 0.10
max_position_per_stock = 0.10 - 0.15
```

### 7.7 均值回归策略参数空间

| 参数                 | 建议搜索范围                       |
| ------------------ | ---------------------------- |
| rsi_period         | [6, 14, 24]                  |
| buy_rsi_threshold  | [20, 25, 30, 35]             |
| sell_rsi_threshold | [45, 50, 60, 70]             |
| boll_period        | [10, 20, 30]                 |
| boll_std           | [1.5, 2.0, 2.5, 3.0]         |
| decline_window     | [3, 5, 10]                   |
| decline_threshold  | [-0.05, -0.08, -0.10, -0.15] |
| stop_loss_ratio    | [0.03, 0.05, 0.08, 0.10]     |
| take_profit_ratio  | [0.05, 0.10, 0.15, 0.20]     |
| max_holding_days   | [5, 10, 20, 30]              |

### 7.8 均值回归策略伪代码

```text
for each trade_date:
    for each symbol in stock_pool:
        if not holding(symbol):
            if rsi < buy_rsi_threshold:
                if close < boll_lower:
                    if recent_return < decline_threshold:
                        if not suspended and not limit_down:
                            generate buy signal
        else:
            if rsi > sell_rsi_threshold:
                generate sell signal
            elif close >= ma20:
                generate sell signal
            elif stop_loss_triggered:
                generate sell signal
            elif max_holding_days_reached:
                generate sell signal
```

## 8. 因子打分策略设计

### 8.1 策略思想

因子打分策略的核心思想是：

> 每个调仓周期对股票池中的股票计算综合评分，买入评分较高的股票，卖出评分下降或不再满足条件的股票。

该策略适合中低频选股和组合管理。

### 8.2 输入数据

因子打分策略需要：

```text
动量因子
反转因子
趋势因子
成交量因子
波动率因子
风险因子
行业信息
当前持仓
账户资金
```

### 8.3 因子组成

第一版可使用：

| 因子               | 说明     |
| ---------------- | ------ |
| return_20d       | 20 日动量 |
| return_60d       | 60 日动量 |
| reversal_5d      | 5 日反转  |
| ma20_ma60_ratio  | 趋势强度   |
| volume_ratio_20d | 成交量变化  |
| volatility_20d   | 波动率惩罚  |
| max_drawdown_20d | 回撤惩罚   |

### 8.4 综合评分公式

示例：

```text
factor_score = 0.25 * momentum_score
             + 0.20 * trend_score
             + 0.15 * volume_score
             + 0.15 * reversal_score
             - 0.15 * volatility_penalty
             - 0.10 * drawdown_penalty
```

权重应可配置和优化。

### 8.5 选股逻辑

每个调仓周期：

1. 计算股票池内所有股票因子分数。
2. 过滤停牌股票。
3. 过滤流动性不足股票。
4. 过滤风险过高股票。
5. 按 factor_score 排序。
6. 选择前 N 只股票。
7. 生成目标持仓。
8. 对不在目标列表中的持仓生成卖出信号。

### 8.6 买入条件

```text
factor_score >= buy_factor_score_threshold
rank <= max_hold_stock_count
is_not_suspended
is_not_limit_up
liquidity_passed
market_filter_passed
```

### 8.7 卖出条件

```text
factor_score < sell_factor_score_threshold
rank > max_hold_stock_count
trigger_stop_loss
trigger_risk_event
```

### 8.8 调仓频率

因子打分策略建议调仓频率：

```text
weekly
monthly
```

不建议每日大规模换仓，避免过高交易成本和过拟合。

### 8.9 因子策略参数空间

| 参数                          | 建议范围               |
| --------------------------- | ------------------ |
| max_hold_stock_count        | [5, 10, 20, 30]    |
| rebalance_frequency         | weekly、monthly     |
| buy_factor_score_threshold  | [0.6, 0.7, 0.8]    |
| sell_factor_score_threshold | [0.3, 0.4, 0.5]    |
| momentum_weight             | [0.1 - 0.4]        |
| trend_weight                | [0.1 - 0.4]        |
| reversal_weight             | [0.0 - 0.3]        |
| volume_weight               | [0.0 - 0.3]        |
| volatility_penalty_weight   | [0.0 - 0.3]        |
| max_position_per_stock      | [0.05, 0.10, 0.15] |

### 8.10 因子策略伪代码

```text
if today is rebalance_day:
    factor_data = get_factor_data_until(today)
    stock_pool = filter_suspended_and_low_liquidity(stock_pool)
    scores = calculate_factor_score(stock_pool)
    target_symbols = select_top_n(scores, max_hold_stock_count)

    for holding_symbol in current_positions:
        if holding_symbol not in target_symbols:
            generate sell signal

    for symbol in target_symbols:
        if not holding(symbol):
            generate buy signal with target_position
        else:
            adjust target_position if needed
```

## 9. 组合型策略设计

### 9.1 策略思想

组合型策略将多个子策略信号进行融合，例如：

```text
趋势跟踪信号 + 因子打分过滤
均值回归信号 + 大盘风险过滤
因子选股 + 趋势确认
```

### 9.2 信号融合方式

支持以下融合方式：

1. AND 规则：多个条件同时满足才买入。
2. OR 规则：任一策略满足即进入候选。
3. 加权评分：多个策略信号转为分数后加权。
4. 主从过滤：主策略生成候选，从策略做过滤。

### 9.3 示例：因子 + 趋势组合策略

买入逻辑：

```text
factor_score 排名前 20
AND
close > MA20
AND
MA20 > MA60
```

卖出逻辑：

```text
factor_score 跌出前 50
OR
close < MA20
OR
触发止损
```

### 9.4 组合策略风险

组合策略容易参数过多，应限制参数数量，防止过拟合。

## 10. 机器学习策略预留设计

### 10.1 设计定位

机器学习策略作为后续扩展，不是第一版核心功能。

机器学习策略可以用于：

2. 预测未来上涨概率。
3. 预测风险等级。
4. 辅助因子打分。

### 10.2 输入特征

可使用：

### 10.3 预测目标

```text
future_return_5d
```

## 11. 仓位生成设计

### 11.1 仓位生成目标

策略输出目标仓位建议，仓位管理模块负责生成最终可执行仓位。

策略层仓位计算目标：

1. 根据信号强度决定基础仓位。
2. 根据策略类型控制风险。
3. 根据大盘状态调整总仓位。
4. 不突破单票和总仓位限制。

### 11.2 基础仓位计算

示例：

```text
target_position = initial_position_ratio * signal_strength_multiplier
```

其中：

```text
signal_strength_multiplier = 0.5 ~ 2.0
```

### 11.3 按风险调整仓位

风险调整公式示例：

```text
risk_adjusted_position = target_position * risk_multiplier
```

risk_multiplier 可由市场状态决定：

| 市场状态 | risk_multiplier |
| ---- | --------------- |
| 强势   | 1.0             |
| 中性   | 0.7             |
| 弱势   | 0.4             |
| 高风险  | 0.0             |

### 11.4 按波动率调整仓位

高波动股票仓位应降低：

```text
vol_adjusted_position = target_position * target_volatility / stock_volatility
```

最终仓位：

```text
final_target_position = min(
    target_position,
    risk_adjusted_position,
    max_position_per_stock
)
```

### 11.5 仓位输出限制

策略层输出仓位不得超过：

```text
max_position_per_stock
```

账户级总仓位限制由仓位管理和风控模块执行。

## 12. 卖出与清仓设计

### 12.1 卖出信号分类

卖出信号分为：

1. 策略卖出。
2. 止损卖出。
3. 止盈卖出。
4. 移动止盈卖出。
5. 到期卖出。
6. 风控卖出。
7. 清仓卖出。

### 12.2 固定止损

当持仓亏损达到阈值：

```text
current_return <= -stop_loss_ratio
```

生成卖出信号。

### 12.3 固定止盈

当持仓盈利达到阈值：

```text
current_return >= take_profit_ratio
```

生成卖出或减仓信号。

### 12.4 移动止盈

记录持仓后的最高价：

```text
highest_price_since_buy
```

当价格从最高点回撤超过阈值：

```text
(close / highest_price_since_buy - 1) <= -trailing_stop_ratio
```

生成卖出信号。

### 12.5 最大持仓天数

当持仓天数超过 max_holding_days，生成卖出信号。

### 12.6 清仓信号

清仓信号可由以下条件触发：

1. 账户最大回撤超过阈值。
2. 大盘进入高风险状态。
3. 策略被人工停止。
4. 风控模块触发全局停止。
5. 策略异常频率触发保护。

## 13. 大盘过滤与市场状态设计

### 13.1 设计目的

大盘过滤用于控制系统性风险。当市场整体处于弱势时，降低策略仓位或停止买入。

### 13.2 市场状态分类

市场状态分为：

```text
strong       强势
neutral      中性
weak         弱势
risk_off     高风险
```

### 13.3 市场状态判断条件

示例：

```text
index_close > index_ma60              strong / neutral
index_close < index_ma60              weak
index_ma20 < index_ma60               weak
index_drawdown_20d > threshold        risk_off
```

### 13.4 市场状态对策略影响

| 市场状态     | 策略行为          |
| -------- | ------------- |
| strong   | 正常买入，正常持仓     |
| neutral  | 正常交易，但仓位适度控制  |
| weak     | 降低新开仓比例       |
| risk_off | 停止买入，必要时减仓或清仓 |

## 15. A 股交易规则对策略的影响

### 15.1 T+1 对策略的影响

A 股普通股票当日买入后不能当日卖出。因此：

1. 策略回测中必须区分总持仓和可卖持仓。
2. 实盘卖出信号必须校验 available_quantity。
3. 当日买入股票即使触发卖出信号，也不能实盘卖出。

### 15.2 涨跌停对策略的影响

1. 涨停时买入可能无法成交。
2. 跌停时卖出可能无法成交。
3. 回测中不能假设所有涨跌停价格都能成交。
4. 实盘下单前必须校验涨跌停价格。

### 15.3 停牌对策略的影响

停牌股票：

1. 不生成买入信号。
2. 持仓股票无法卖出。
3. 回测中持仓市值可使用最近可用价格估值。

### 15.4 交易单位对策略的影响

A 股买入通常应按 100 股整数倍。策略输出目标仓位后，仓位管理模块需要转换为合规股数。

## 16. 回测、模拟盘、实盘一致性设计

### 16.1 统一策略逻辑

同一策略在回测、模拟盘和实盘中应使用相同的信号生成逻辑。

差异应只存在于：

1. 数据来源。
2. 成交方式。
3. 订单状态。
4. 是否真实下单。

### 16.2 三种模式的数据来源

| 模式  | 数据来源           |
| --- | -------------- |
| 回测  | 历史行情、历史指标、历史因子 |
| 模拟盘 | 实时或准实时行情、模拟账户  |
| 实盘  | 实时或准实时行情、真实账户  |

### 16.3 三种模式的输出

| 模式  | 输出                  |
| --- | ------------------- |
| 回测  | 模拟交易记录、资产曲线、回测报告    |
| 模拟盘 | 模拟订单、模拟成交、模拟持仓、模拟报告 |
| 实盘  | 交易计划、风控结果、真实订单、真实成交 |

### 16.4 一致性要求

1. 策略参数版本一致。
2. 指标计算方式一致。
3. 信号生成逻辑一致。
4. 成本模型尽量一致。
5. A 股交易规则处理一致。
6. 风控参数尽量一致。

## 17. 防未来函数设计

### 17.1 未来函数定义

未来函数是指策略在某个历史时点使用了当时不可能获得的信息，例如：

1. 使用未来收盘价生成当日买入信号。
2. 使用未来财务数据。
3. 使用未来指数成分股。
4. 使用未来涨跌停状态进行提前判断。
5. 使用未来成交结果优化当前交易。

### 17.2 防未来函数规则

系统应遵守：

1. 策略只能读取 current_date 及之前的数据。
2. 使用收盘价生成信号时，默认次日执行。
3. 财务数据必须考虑披露日期。
4. 指标计算必须使用历史滚动窗口。
5. 因子计算不得使用未来收益作为特征。
6. 回测成交不能使用信号生成之后才知道的价格，除非规则明确为下一交易日成交。

### 17.3 默认成交时点

推荐默认规则：

```text
T 日收盘后生成信号
T+1 日开盘或指定时间执行
```

这样可以减少未来函数风险。

### 17.4 防未来函数测试

测试方法包括：

1. 随机截断历史数据，验证策略只使用截断前数据。
2. 对比不同数据窗口结果。
3. 检查指标 rolling 是否包含未来数据。
4. 检查信号日期和成交日期是否合理。

## 18. 防过拟合设计

### 18.1 过拟合风险

策略过拟合表现为：

1. 样本内收益很高，样本外收益很差。
2. 参数极端。
3. 交易次数过少但收益很高。
4. 换手率极高。
5. 对某几个股票或某一段行情依赖严重。

### 18.2 防过拟合措施

系统应支持：

1. 样本内训练。
2. 样本外验证。
3. 滚动窗口验证。
4. 多股票池验证。
5. 多市场环境验证。
6. 参数范围约束。
7. 换手率惩罚。
8. 最大回撤惩罚。
9. 集中度惩罚。
10. 交易频率惩罚。

### 18.3 样本划分建议

示例：

```text
2018-2021：训练集
2022-2023：验证集
2024-2025：测试集
```

### 18.4 策略评分不只看收益

默认综合评分：

```text
score = annual_return * 0.35
      + sharpe_ratio * 0.25
      - max_drawdown * 0.25
      - turnover_penalty * 0.10
      - concentration_penalty * 0.05
```

后续可以加入：

```text
frequency_penalty
liquidity_penalty
stability_score
out_sample_score
```

## 19. 策略频率限制设计

### 19.1 设计目的

虽然系统不做高频交易，但仍需要防止策略因 bug 或错误配置短时间内产生大量信号或订单。

### 19.2 频率限制规则

系统应支持：

1. 单策略每日最大信号数。
2. 单策略每日最大订单数。
3. 单股票每日最大交易次数。
4. 最短信号间隔。
5. 最小调仓周期。
6. 单日最大换手率。

### 19.3 异常频率处理

当策略触发异常频率时：

1. 停止生成新交易计划。
2. 暂停策略实例。
3. 记录风控事件。
4. 发送告警。
5. 等待人工确认。

### 19.4 频率限制示例

```json
  "min_signal_interval_minutes": 60,

```

## 23. 策略测试设计

### 23.1 单元测试

需要测试：

1. 参数校验。
2. 指标读取。
3. 买入信号生成。
4. 卖出信号生成。
5. 清仓信号生成。
6. 目标仓位计算。
7. 信号原因生成。
8. 频率限制。

### 23.2 回测测试

需要测试：

1. 策略在固定数据上的输出是否稳定。
2. 是否存在未来函数。
3. 买卖信号是否符合规则。
4. T+1 是否正确处理。
5. 涨跌停是否正确处理。
6. 手续费是否正确扣除。

### 23.3 模拟盘测试

需要测试：

1. 策略是否按预定频率运行。
2. 交易计划是否正确生成。
3. 模拟盘风控是否生效。
4. 模拟订单和成交是否正确。

### 23.4 实盘前测试

需要测试：

1. 策略信号是否能追溯到订单。

3. 异常高频信号会暂停策略。
4. 账户同步失败时不会生成实盘订单。
5. 手动停止策略后不会继续生成交易计划。

## 24. 策略上线流程设计

### 24.1 上线阶段

策略上线应经过以下阶段：

```text
策略开发
   ↓
本地单元测试
   ↓
历史回测
   ↓
参数优化
   ↓
样本外验证
   ↓
模拟盘运行
   ↓
实盘候选评审
   ↓
小资金实盘
   ↓
正式实盘
```

### 24.2 上线前条件

策略进入实盘前必须满足：

1. 回测结果完整。
2. 样本外验证通过。
3. 模拟盘运行稳定。
4. 最大回撤在可接受范围内。
5. 换手率在可接受范围内。
6. 策略频率符合低频定位。
7. 风控参数已配置。
8. 实盘交易开关明确开启。
9. 小资金灰度测试通过。

### 24.3 下线条件

策略应在以下情况下暂停或下线：

1. 实盘亏损超过阈值。
2. 账户最大回撤超过阈值。
3. 策略连续亏损超过阈值。
4. 策略触发异常高频信号。
5. 策略逻辑出现异常。
6. 策略与回测表现严重偏离。
7. 人工判断需要停止。

## 26. 与其他模块的关系

### 26.1 与数据模块关系

策略从数据模块读取行情、交易日历、停牌和涨跌停状态。

### 26.2 与指标模块关系

策略依赖指标模块提供均线、MACD、RSI、BOLL、ATR 等指标。

### 26.3 与因子模块关系

因子打分策略依赖因子模块提供因子数据和综合评分。

### 26.4 与回测模块关系

回测模块调用策略生成历史信号，并模拟成交和持仓变化。

### 26.5 与参数优化模块关系

参数优化模块通过多次调用回测模块评估不同策略参数。

### 26.6 与模拟盘模块关系

模拟盘模块按实际交易日调度策略运行，并生成模拟订单。

### 26.7 与实盘交易模块关系

实盘模式下，策略只生成交易计划，不直接下单。实盘订单由订单模块和券商接口模块处理。

### 26.8 与风控模块关系

策略信号必须经过风控模块，风控模块可以拒绝、降仓、暂停或清仓。

## 27. 总结

本文档完成了 A 股低频量化策略研究、回测、模拟盘与实盘执行系统的策略设计。

系统策略体系以低频交易为长期定位，支持趋势跟踪、均值回归、因子打分和组合型策略。策略模块只生成信号和目标仓位，不直接下单，不直接修改资金和持仓。所有实盘交易必须经过交易计划、仓位管理、风控检查和订单管理模块。
